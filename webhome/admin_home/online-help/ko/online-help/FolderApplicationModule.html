새 어플리케이션 모듈 배치

네비게이션 경로: Home > node-name > 어플리케이션 모듈 디플로이

개요

이 페이지에서는 작성한 Java EE 어플리케이션 또는 모듈들을 JEUS 서버에 디플로이할 수 있는 기능을 제공한다. JEUS에서 디플로이에 익숙치 않은 사용자를 위해 디플로이 과정을 wizard 형식으로 제공하고 있다. 순서에 따라 설정하고자 하는 값들을 설정하여 주면 보다 쉽게 어플리케이션을 디플로이 할 수 있다.

디플로이 순서는 다음과 같다.

  1. 모듈 선택 : 디플로이 하고자 하는 어플리케이션 또는 모듈을 선택한다.

  2. 대상 선택 : 어플리케이션 또는 모듈이 디플로이 될 대상(노드, 컨테이너 등)을 선택한다.

  3. 선택 사항 : 각 컴포넌트별 설정 및 어플리케이션 또는 모듈에 공통적으로 적용할 값들을 설정을 한다.

  4. 배치 : 1-3의 과정에서 설정한 값을 바탕으로 어플리케이션 또는 모듈을 디플로이 한다.

  5. 결과 : 디플로이 결과를 출력한다.

각 과정에 대한 자세한 설명은 아래의 절에서 설명한다.

모듈 선택

이 단계에서는 디플로이 하고자 하는 어플리케이션 또는 모듈을 선택한다. 여기서는 모듈을 선택하는 방법을 크게 3가지 형태(이 페이지에서는 각각의 탭-모듈, 절대 경로, 파일 업로드-으로 분리되어 있다.)로 제공한다.

반드시 하나 이상의 어플리케이션 또는 모듈을 선택해야만 다음 단계(대상 선택)로 넘어가며, 아무것도 선택하지 않았을 경우에는 '배치할 모듈을 선택해야 합니다.'라는 안내 메세지가 출력된다.

모듈

주어진 디렉토리 또는 탐색한 디렉토리 내에 존재하는 어플리케이션을 선택하는 두 가지 방법을 제공한다.(서브탭으로 분리)

  1. 어플리케이션 홈 : 어플리케이션 홈(JEUS_HOME\webhome\app_home)에 위치한 어플리케이션을 선택할 수 있다. 이 탭에서는 어플리케이션 홈에 위치한 어플리케이션들의 정보를 표로 보여준다. 표에 나타난 정보는 아래에서 설명한다.

  2. 디렉토리 선택 : 어플리케이션 홈 이외의 디렉토리를 탐색하여, 그 디렉토리에 존재하는 어플리케이션을 선택할 수 있다. 표 위에 디렉토리 경로로 표시된 것이 디텍토리 탐색기이다. 디렉토리의 구조에 따라 탐색할 수 있다. 이 탭에서도 마찬가지로 해당 디렉토리에 존재하는 어플리케이션들의 정보를 표로 보여준다. 표에 나타난 정보는 아래에서 설명한다.

  • 표에 나타난 어플리케이션 정보(첫번째 칼럼부터 순서대로 설명)

    • 어플리케이션 또는 모듈을 선택할 수 있는 선택 상자가 있다. 모듈의 복수 선택이 가능하며, 마지막 칼럼에 가 표시되어 있는 경우에는 선택 상자가 비활성화된다. 즉, 이미 디플로이된 어플리케이션은 WebAdmin을 통해 다시 디플로이 명령을 실행할 수 없다.

    • 모듈 이름 : 어플리케이션 또는 모듈의 종류에 다른 아이콘과 어플리케이션 이름을 출력한다. 어플리케이션 이름은 텍스트 상자 안에 표기되며, 이 이름은 사용자가 변경할 수 있다. 변경한 경우, 변경한 이름으로 어플리케이션 이름을 설정하여 디플로이 한다.

      • : Java EE 어플리케이션 (.ear 파일)

      • : EJB 모듈 (.jar 파일)

      • : 웹 어플리케이션 모듈 (.war 파일)

      • : 어플리케이션 클라이언트 모듈 (.jar 파일)

      • : 리소스 어댑터 모듈 (.rar 파일)

    • 경로 : 어플리케이션의 경로를 보여준다. 사용자가 모듈을 선택할 때, 선택한 모듈이 올바른 경로에 있는 것인지 확인할 수 있다.

    • 이미 배치된 어플리케이션의 경우 로 표시하여 알려준다. 표시된 어플리케이션은 선택할 수 없게 되어 있으므로, 또 다시 디플로이 할 수 없다.

절대 경로

두 가지 경우에 절대 경로를 사용할 수 있다.

  1. 임의의 디렉토리에 존재하는 어플리케이션을 디플로이 하고자 할 때

  2. 디렉토리에 대한 오토 디플로이 설정을 하고자 할 때

두 경우에 대해 사용자가 직접 디렉토리 경로를 입력할 수 있다. 절대 경로 텍스트 입력란에 경로를 입력하면, 그 경로가 로컬에 존재하고 디렉토리인 경우에는 autodeploy 설정을 할 수 있도록 체크 박스가 나타난다. 이것을 설정하면 해당 디렉토리를 오토 디플로이로 설정하여, 디렉토리 내 모든 어플리케이션에 대해 오토 디플로이를 실행한다. 즉, 어플리케이션이 추가되면 자동으로 디플로이를 실행하고, 삭제되면 언디플로이를 실행한다.

대상 선택

이 단계에서는 어플리케이션을 디플로이 할 대상을 선택한다. 대상의 기본적인 단위는 엔진 컨테이너이므로 엔진 컨테이너 단위로 선택할 수 있다.

디플로이의 대상에는 다음의 3가지가 있다.

  • 현재 노드 : 현재 노드의 모든 엔진 컨테이너가 디플로이 대상이 될 수 있다.

  • 클러스터 : 현재 노드와 클러스터 된 모든 노드의 엔진 컨테이너가 디플로이 대상이 될 수 있다.

  • 가상 호스트 : 가상 호스트 내에 모든 엔진 컨테이너가 디플로이 대상이 될 수 있다. 가상 호스트 설정에 대한 내용은 가상 호스트을 참고하기 바란다.

웹 어플리케이션 모듈인 경우에는 컨텍스트 그룹과 가상 호스트를 설정할 수 있다(2. 가상 호스트 페이지 참고). 각 엔진 컨테이너 별로 서블릿 엔진의 설정 파일(WebMain.xml)에 설정된 컨텍스트 그룹과 가상 호스트 목록이 나타난다. 가상 호스트의 경우에는 설정이 되어 있지 않은 경우 'Default Virtual Host'가 목록에 나타난다.

가능한 대상에는 사용자가 선택할 수 있는 디플로이 대상이 나타나므로, 선택할 대상에 대해 선택 상자를 클릭하면 그 대상이 선택된다.

  • 모두 : 클러스터, 가상 호스트를 포함한 모든 가능한 노드 내에 있는 엔진 컨테이너가 모두 선택된다.

  • 노드 : 클러스터, 가상 호스트를 포함한 모든 가능한 노드를 표시한다. 아이콘과 함께 노드 이름이 표시된다.

  • 엔진 컨테이너 : 클러스터, 가상 호스트를 포함한 각 노드에 설정된 엔진 컨테이너를 표시한다. 아이콘과 함께 엔진 컨테이너 이름이 표시된다. 웹 어플리케이션 모듈인 경우에는 컨테이너 하위에 컨텍스트 그룹과 가상 호스트 설정 목록이 추가로 나타난다.

선택 사항

이 페이지에서는 각 컴포넌트별 설정 및 어플리케이션 또는 모듈에 공통적으로 적용할 값들을 설정을 한다.

컴포넌트별 선택 사항

각 어플리케이션 또는 모듈에 포함된 컴포넌트 별 설정을 할 수 있다. 화면에서는 각 컴포넌트 별로 별도의 탭 화면으로 나누어져 있다. 탭의 상위에는 각 컴포넌트의 타입을 구분하는 아이콘(모듈 아이콘과 동일)과 컴포넌트의 이름이 표시된다. 원하는 탭을 선택하여 해당 컴포넌트 설정을 할 수 있다.

EJB 컴포넌트 선택 사항

이 어플리케이션 내 EJB 컴포넌트에 대한 특별한 설정을 하고 싶을때 사용한다.

Uri

어플리케이션 내에 있는 이 EJB 컴포넌트에 해당하는 uri를 설정한다. standalone 모듈인 경우에는 이 값을 설정할 필요가 없다.

WebAdmin에서는 이 값을 자동으로 설정해준다.

Client View Path

이 EJB 컴포넌트의 스텁 클래스들이 존재할 디렉토리를 설정한다.

Keep Generated

이 EJB 컴포넌트의 스텁 클래스들을 생성할때 자바 소스 파일을 남길지의 여부를 지정한다. 이 설정이 없으면 <application> 하위의 <keep-generated>설정을 사용한다.

Java Security Permission

이 EJB 컴포넌트에 J2SE security permission을 설정할 수 있다.

Description

이 컴포넌트에 security permission 설정에 대한 설명을 적는다.

Security Permission Spec

이 컴포넌트에 적용할 security permission 설정을 기술한다. 형식은 java policy file과 동일하다.

공통 선택 사항

각 어플리케이션 또는 모듈에 공통적으로 적용될 값을 설정할 수 있다.

Fast Deploy

해당 어플리케이션 내의 EJB module, 웹 어플리케이션의 웹 서비스 모듈에 대해 패스트 디플로이(fast deploy)를 적용하도록 설정한다. 즉, 이 어플리케이션이 디플로이 과정에서 생성되어야 하는 클래스를 미리 포함하고 있다고 가정한다. 디플로이를 처음 하거나 모듈의 클래스 파일들이 변경되었을 경우에는 이 옵션을 사용하지 않는다. 그러나 appcompiler 툴을 사용하여 클래스들을 생성한 모듈의 경우에는 이 옵션을 사용할 수 있다.

Deploy When Booting

JEUS가 부팅될 때 해당 어플리케이션을 디플로이 할 것인지에 대한 설정을 한다. 기본값은 true이다.

Classloading

이 application이 사용할 classloading 방식을 선택한다. 지정하지 않으면 jeus.classloading system property에 설정되어 있는 값을 사용한다. JEUS 6에서는 ISOLATED 방식을 사용할 것을 적극 권장한다.

Class Ftp Unit

어플리케이션에 포함된 EJB 모듈의 클래스를 class ftp를 통해서 클라이언트(client)에게 전달할 때의 단위로 CLASS와 JAR 두가지 단위가 있다. 기본값은 JAR이다.

Security Domain Name

이 application에게 적용할 security domain을 설정한다. 지정하지 않으면 DEFAULT_APPLICATION_DOMAIN을 사용하게 된다.

Keep Generated

이 applicaton 내의 module들에 keep-generated를 적용한다. 즉, 이 appliation이 deploy과정에서 generated되어야 하는 클래스를 미리 포함하고 있다고 가정한다. 만약 설정이 되어 있지 않으면 jeus.application.keepgenerated system property에 지정된 값이 사용된다.

Auto Deploy

모듈 및 디렉토리에 대한 auto deploy 설정을 한다.

이 어플리케이션에 대해 archive 파일이나 Java EE 표준 DD가 변경되었을 경우 자동적으로 리디플로이(redeploy)를 실행한다.

Check Absolute Path

디렉토리에 대한 오토 디플로이 설정을 할 경우에만 사용된다. 어플리케이션에 대한 오토 디플로이 설정을 하는 경우에는 이 태그의 값을 false로 해야한다. 설정을 하지 않으면, 자동으로 false가 설정된다.

Auto Deploy Check Interval

JEUS 서버가 해당 경로에 있는 어플리케이션들의 변경 사항을 확인하는 주기를 설정한다. 기본값은 10초이며, 주기가 너무 짧을 경우 성능 저하가 있을 수 있다는 것을 유념하기 바란다.

디플로이

이 페이지에서는 디플로이 옵션은 설정하고, 어플리케이션 디플로이를 실행한다.

디플로이 옵션 선택은 런타임 디플로이가 실행될 때(엔진 컨테이너가 부팅되어 있을 때)만 선택할 수 있으며, 엔진 컨테이너가 다운된 상태에서는 JEUSMain.xml에 어플리케이션을 등록하는 영구적인 디플로이만 실행한다.

  1. 런타임 디플로이 : 런타임 디플로이를 선택하면 디플로이를 바로 실행할 수 있다.

  2. 영구적인 디플로이 : [모듈 선택]~[선택 사항]까지 설정한 값들로 JEUSMain.xml에 이 어플리케이션을 등록한다. 엔진컨테이너가 다운된 상태에서는 오직 영구적인 디플로이만을 진행할 수 있다.

결과

디플로이를 진행하다가 문제가 생겼을 경우, 이 페이지에서 그 에러 상황(Exception Stack Trace)에 대해서 확인할 수 있다.
