핸들러

개요

이 페이지에서는 로거에 새로운 핸들러를 추가하거나, 이미 로거에 등록된 핸들러의 설정을 변경 또는 삭제할 수 있다.

새 로그 핸들러 생성

새 로그 핸들러 생성 - 콘솔 핸들러

새로운 콘솔 핸들러를 로거에 추가한다. 콘솔 핸들러는 로거를 통해 로깅되는 로그 메세지를 콘솔 화면에 출력한다.

핸들러 종류

현재 설정을 변경하고자 하는 핸들러의 종류를 보여준다.

Name

이 핸들러의 이름으로, 핸들러의 로거에 대해서만 유일하게(unique) 설정되면 된다.

Level

이 핸들러의 레벨로, 로거를 통과한 메세지의 레벨이 이 핸들러 레벨에 포함될 경우에만 이 핸들러에 의해 출력된다. 예를 들어, 로거의 레벨이 INFO이고 이 로거에 설정된 핸들러의 레벨이 FINE이라면, 이 핸들러에 의해서는 INFO 레벨까지만 처리될 수 있다. 핸들러에서 FINE 레벨까지 로깅을 하고 싶지만, 이 레벨의 메세지들은 로거를 통과하지 못하므로, 핸들러에서 CONFIG와 FINE 레벨의 메세지는 처리할 수 없는 것이다.

레 벨설 명
FATALJEUS 4.x대의 logger의 level로 compatibility를 위해 지원한다.SEVERE에 해당하는 레벨
NOTICEWARNING에 해당하는 레벨
INFORMATIONINFO에 해당하는 레벨
DEBUGFINE에 해당하는 레벨
SEVERE치명적인 실패, 오류(serious failure)에 대한 로깅 레벨
WARNING잠재적인 문제(potential problem)에 대한 로깅 레벨
INFO안내 메세지(information message)에 대한 로깅 레벨
CONFIG설정 메세지(static configuration message)에 대한 로깅 레벨
FINE트레이스(trace) 정보에 대한 로깅 레벨
FINERFINE 레벨보다 좀더 자세한 트레이스(trace) 정보에 대한 로깅 레벨
FINEST가장 자세한 트레이스(trace) 정보에 대한 로깅 레벨
ALL모든 메세지에 대한 로깅 레벨

File Name

이 핸들러가 사용할 이름으로, 로거를 통해 로깅되는 로그 메세지를 여기에서 지정한 파일에 저장한다.

만약 사용자가 파일 이름을 설정하지 않는 경우, 각 jeus 로거의 로그 메세지는 정해진 위치에 파일을 생성한다. 이 때 각 엔진의 로그 파일은 각 엔진에 로깅 설정을 했을 시에만 생성된다. 다음은 각 로거의 기본 위치이다.

  • 디폴트 컨테이너 사용할 경우

    • jeus : JEUS manager에서 남겨지는 로그는 JEUS_HOME\logs\<node-name> 디렉토리에 JeusServer.log라는 파일로 남겨진다. 각 엔진 컨테이너의 로그는 JEUS manager에서 생성한 jeus 로거를 사용하여 로그를 남긴다.

    • jeus.systemuser : 각 엔진 컨테이너의 user log는 JEUS_HOME\logs\<node-name>\SYSTEM_ENGINE\<node-name>_<container-name> 디렉토리에 UserLog.log 라는 파일로 남겨진다.

    • jeus.ejb : EJB 엔진의 로거는 실행되는 엔진 컨테이너 이름의 JEUS_HOME\logs\<node-name>\SYSTEM_ENGINE\ejb 디렉토리 아래에 error.log라는 파일로 남겨진다.

    • jeus.servlet : 서블릿 엔진의 로거는 실행되는 엔진 컨테이너 이름의 JEUS_HOME\logs\<node-name>\SYSTEM_ENGINE\servlet 디렉토리 아래에 error.log라는 파일로 남겨진다.

    • jeus.jms : JMS 엔진의 로거는 실행되는 엔진 컨테이너 이름의 JEUS_HOME\logs\<node-name>\SYSTEM_ENGINE\jms 디렉토리 아래에 error.log라는 파일로 남겨진다.

  • 디폴트 컨테이너를 사용하지 않을 경우

    • jeus : JEUS manager에서 남겨지는 로그는 JEUS_HOME\logs\<node-name>\<node-name>_<container-name> 디렉토리에 <node-name>_<container-name>.log라는 파일로 남겨진다. 각 엔진 컨테이너의 로그는 JEUS_HOME\logs\<node-name>\<node-name>_<container-name> 디렉토리에 <node-name>_<container-name>.log 라는 파일로 남겨진다.

    • jeus.systemuser : 각 엔진 컨테이너의 user log는 JEUS_HOME\logs\<node-name>\<node-name>_<container-name> 디렉토리에 UserLog.log 라는 파일로 남겨진다.

    • jeus.ejb : EJB 엔진의 로거는 실행되는 엔진 컨테이너 이름의 JEUS_HOME\logs\<node-name>\<node-name>_<container-name>\ejb 디렉토리 아래에 error.log라는 파일로 남겨진다.

    • jeus.servlet : 서블릿 엔진의 로거는 실행되는 엔진 컨테이너 이름의 JEUS_HOME\logs\<node-name>\<node-name>_<container-name>\servlet 디렉토리 아래에 error.log라는 파일로 남겨진다.

    • jeus.jms : JMS 엔진의 로거는 실행되는 엔진 컨테이너 이름의 JEUS_HOME\logs\<node-name>\<node-name>_<container-name>\jms 디렉토리 아래에 error.log라는 파일로 남겨진다.

Enable Rotation

이 핸들러가 사용하는 파일을 rotation 할 것인지 여부를 정한다. 로그파일 rotation이란 하나의 파일에만 로깅하고 설정한 시간이 지나거나 현재 로깅하고 있는 파일이 설정한 파일 사이즈가 되면 이전에 로깅하던 파일은 새로운 이름으로 바꾸고 새로 로깅할 파일은 이전에 로깅하던 파일이름으로 계속 로깅하는 것이다. 기본값은 false이다.

Rotation Count

이 핸들러가 사용하는 파일이 rotation기능을 사용할 때만 의미가 있다. rotation된 파일이 디렉토리에 저장될 수 있는 개수를 설정한다. 이 값을 설정하지 않으면 파일 사이즈로 rotation하는 경우에는 99999개 까지 쌓이고, 날짜나 시간으로 rotation하는 경우에는 rotation된 파일은 계속 쌓인다.

Rotation Dir

이 핸들러가 사용하는 파일이 rotation기능을 사용할 때만 의미가 있다. rotation된 백업파일이 저장될 디렉토리를 설정한다. 이 값을 설정하지 않으면 기본적으로 현재 로깅하고 있는 디렉토리에 rotation한다.

Valid Day, Valid Hour, Valid Size

이 핸들러가 사용하는 파일을 여기에서 설정된 기간동안이나 사이즈만큼만 사용하고 계속 갱신할 경우에 사용한다. 날짜 또는 시간 또는 사이즈단위로 파일을 바꿀 때 사용한다. 이 경우에는 핸들러가 사용하는 파일 이름 뒤에 날짜 또는 날짜와 시간 또는 일정 사이즈가 되면 증가하는 인덱스가 자동으로 붙여진다.

Enable Rotation을 사용하면서 이 값들을 설정하지 않으면 기본적으로 Valid Size가 1024kb로 동작한다.

Buffer Size

이 핸들러가 파일에 로깅할 때 사용하는 버퍼의 크기를 설정한다.

Append

이 핸들러가 로깅을 저장하기 위해 설정된 파일 이름이 중복되는 경우, 그 파일에 계속해서 로그를 기록할 것인지를 설정한다. false인 경우에는 기존의 파일이 삭제되고, 그 이름의 새로운 파일을 생성하여 기록한다.

만약 Enable Rotation이 설정되어 있다면 Append의 값이 true인 경우는 서버를 재부팅할 때 이전에 로깅하던 파일에 계속해서 기록하고 false인 경우 서버를 재부팅을 할 때 새로운 파일을 생성하여 로그를 기록한다.

Smtp Host Address

이메일을 보낼 호스트(host)의 주소를 지정한다.

From Address

이메일을 보내는 사람의 주소를 지정한다.

To Address

이메일을 받는 사람의 주소를 지정한다.

Cc Address

이메일을 참조로 받는 사람의 주소를 지정한다.

Bcc Address

이메일을 숨은 참조로 받는 사람의 주소를 지정한다.

Send For All Messages

모든 메세지를 SMTP 핸들러로 보낼지를 결정한다. 만약 false이면 JEUS system에서 이메일로 전송하기로 결정되어 있는 메세지만 이 핸들러를 사용해서 보내진다. 현재 이 설정은 jeus.systemuser 로거에만 유효하다.

File Name

이 핸들러가 사용할 이름으로, 로거를 통해 로깅되는 로그 메세지를 여기에서 지정한 파일에 저장한다.

만약 사용자가 파일 이름을 설정하지 않는 경우, 각 jeus 로거의 로그 메세지는 정해진 위치에 파일을 생성한다. 이 때 각 엔진의 로그 파일은 각 엔진에 로깅 설정을 했을 시에만 생성된다. 다음은 각 로거의 기본 위치이다.

  • 디폴트 컨테이너 사용할 경우

    • jeus : JEUS manager에서 남겨지는 로그는 JEUS_HOME\logs\<node-name> 디렉토리에 JeusServer.log라는 파일로 남겨진다. 각 엔진 컨테이너의 로그는 JEUS manager에서 생성한 jeus 로거를 사용하여 로그를 남긴다.

    • jeus.systemuser : 각 엔진 컨테이너의 user log는 JEUS_HOME\logs\<node-name>\SYSTEM_ENGINE\<node-name>_<container-name> 디렉토리에 UserLog.log 라는 파일로 남겨진다.

    • jeus.ejb : EJB 엔진의 로거는 실행되는 엔진 컨테이너 이름의 JEUS_HOME\logs\<node-name>\SYSTEM_ENGINE\ejb 디렉토리 아래에 error.log라는 파일로 남겨진다.

    • jeus.servlet : 서블릿 엔진의 로거는 실행되는 엔진 컨테이너 이름의 JEUS_HOME\logs\<node-name>\SYSTEM_ENGINE\servlet 디렉토리 아래에 error.log라는 파일로 남겨진다.

    • jeus.jms : JMS 엔진의 로거는 실행되는 엔진 컨테이너 이름의 JEUS_HOME\logs\<node-name>\SYSTEM_ENGINE\jms 디렉토리 아래에 error.log라는 파일로 남겨진다.

  • 디폴트 컨테이너를 사용하지 않을 경우

    • jeus : JEUS manager에서 남겨지는 로그는 JEUS_HOME\logs\<node-name>\<node-name>_<container-name> 디렉토리에 <node-name>_<container-name>.log라는 파일로 남겨진다. 각 엔진 컨테이너의 로그는 JEUS_HOME\logs\<node-name>\<node-name>_<container-name> 디렉토리에 <node-name>_<container-name>.log 라는 파일로 남겨진다.

    • jeus.systemuser : 각 엔진 컨테이너의 user log는 JEUS_HOME\logs\<node-name>\<node-name>_<container-name> 디렉토리에 UserLog.log 라는 파일로 남겨진다.

    • jeus.ejb : EJB 엔진의 로거는 실행되는 엔진 컨테이너 이름의 JEUS_HOME\logs\<node-name>\<node-name>_<container-name>\ejb 디렉토리 아래에 error.log라는 파일로 남겨진다.

    • jeus.servlet : 서블릿 엔진의 로거는 실행되는 엔진 컨테이너 이름의 JEUS_HOME\logs\<node-name>\<node-name>_<container-name>\servlet 디렉토리 아래에 error.log라는 파일로 남겨진다.

    • jeus.jms : JMS 엔진의 로거는 실행되는 엔진 컨테이너 이름의 JEUS_HOME\logs\<node-name>\<node-name>_<container-name>\jms 디렉토리 아래에 error.log라는 파일로 남겨진다.

Enable Rotation

이 핸들러가 사용하는 파일을 rotation 할 것인지 여부를 정한다. 로그파일 rotation이란 하나의 파일에만 로깅하고 설정한 시간이 지나거나 현재 로깅하고 있는 파일이 설정한 파일 사이즈가 되면 이전에 로깅하던 파일은 새로운 이름으로 바꾸고 새로 로깅할 파일은 이전에 로깅하던 파일이름으로 계속 로깅하는 것이다. 기본값은 false이다.

Rotation Count

이 핸들러가 사용하는 파일이 rotation기능을 사용할 때만 의미가 있다. rotation된 파일이 디렉토리에 저장될 수 있는 개수를 설정한다. 이 값을 설정하지 않으면 파일 사이즈로 rotation하는 경우에는 99999개 까지 쌓이고, 날짜나 시간으로 rotation하는 경우에는 rotation된 파일은 계속 쌓인다.

Rotation Dir

이 핸들러가 사용하는 파일이 rotation기능을 사용할 때만 의미가 있다. rotation된 백업파일이 저장될 디렉토리를 설정한다. 이 값을 설정하지 않으면 기본적으로 현재 로깅하고 있는 디렉토리에 rotation한다.

Valid Day, Valid Hour, Valid Size

이 핸들러가 사용하는 파일을 여기에서 설정된 기간동안이나 사이즈만큼만 사용하고 계속 갱신할 경우에 사용한다. 날짜 또는 시간 또는 사이즈단위로 파일을 바꿀 때 사용한다. 이 경우에는 핸들러가 사용하는 파일 이름 뒤에 날짜 또는 날짜와 시간 또는 일정 사이즈가 되면 증가하는 인덱스가 자동으로 붙여진다.

Enable Rotation을 사용하면서 이 값들을 설정하지 않으면 기본적으로 Valid Size가 1024kb로 동작한다.

Buffer Size

이 핸들러가 파일에 로깅할 때 사용하는 버퍼의 크기를 설정한다.

Append

이 핸들러가 로깅을 저장하기 위해 설정된 파일 이름이 중복되는 경우, 그 파일에 계속해서 로그를 기록할 것인지를 설정한다. false인 경우에는 기존의 파일이 삭제되고, 그 이름의 새로운 파일을 생성하여 기록한다.

만약 Enable Rotation이 설정되어 있다면 Append의 값이 true인 경우는 서버를 재부팅할 때 이전에 로깅하던 파일에 계속해서 기록하고 false인 경우 서버를 재부팅을 할 때 새로운 파일을 생성하여 로그를 기록한다.
