개요

이 페이지에서는 JEUS내에 존재하는 모든 쓰레드 풀 목록을 표의 형태로 볼 수 있고, 쓰레드 풀의 이름으로 검색도 가능하다. 매니저, 컨테이너별로 현재 사용하고 있는 통계 가능한 모든 쓰레드 풀에 대해 아래와 같은 정보를 기본적으로 보여주며, 쓰레드 풀 이름을 클릭하면 정보 페이지로 이동하여 좀 더 상세한 정보와 기존 쓰레드 풀의 설정을 수정할 수 있다. 우측 하단에 존재하는 "새로고침" 버튼을 클릭하면 실시간으로 변경된 쓰레드 풀의 정보를 업데이트 할 수 있다.

Name

해당 쓰레드 풀을 식별하는 이름

Core Size

해당 쓰레드 풀에 설정된 core pool size의 값

Max Size

해당 쓰레드 풀에 설정된 max pool size의 값. 허용 가능한 최대치의 쓰레드 갯수를 의미한다.

Keep Alive Time

해당 쓰레드 풀에 설정된 keep alive time(단위 milli-seconds)의 값

Current Size

해당 쓰레드 풀에서 현재 유지하고 있는 쓰레드의 갯수

Active

해당 쓰레드 풀에서 활성화된 쓰레드의 갯수

Task

해당 쓰레드 풀의 task 정보. "Scheduled", "Completed" 두가지가 있으며, 전자의 경우 현재까지 스케줄링된 task의 갯수를 나타내며, 후자의 경우 스케줄링되어 현재 완료한 task의 갯수를 나타낸다.

Execution Time

해당 쓰레드 풀의 현재까지 실행된 쓰레드 수행시간(단위 milli-seconds). "Min", "Mean", "Max" 세가지가 있으며, 각각 최소의 수행시간, 평균 수행시간, 최대 수행시간을 나타낸다.

정보

이 페이지에서는 해당 쓰레드 풀의 좀 더 상세한 정보를 보여주며, 기존 설정에 대해 수정, 변경 할 수 있는 기능 및 해당 쓰레드 풀에서 관리되고 있는 쓰레드들의 상세정보와 특정 쓰레드에 대한 StackTrace정보 및 Interrupt 컨트롤 기능을 제공한다. 크게 두가지로 정보를 표시하여 나타내는데, 첫번째는 쓰레드 풀의 상세 정보를 보여주고, 그 아래로는 해당 쓰레드 풀이 관리하고 있는 쓰레드들에 대한 상세 정보를 표로 보여준다.

Name

해당 쓰레드 풀을 식별하는 이름

Core Size

해당 쓰레드 풀에 설정된 core pool size의 값. 기존 설정 값이 기본적으로 보여지며, 이 값을 동적으로 수정할 수 있다.

Max Size

해당 쓰레드 풀에 설정된 max pool size의 값. 허용 가능한 최대치의 쓰레드 갯수를 의미한다. 기존 설정 값이 기본적으로 보여지며, 이 값을 동적으로 수정할 수 있다.

Keep Alive Time

해당 쓰레드 풀에 설정된 keep alive time(단위 milli-seconds)의 값. 기존 설정 값이 기본적으로 보여지며, 이 값을 동적으로 수정할 수 있다.

Allows CoreThread TimeOut

core 쓰레드들에 대해 keep alive time의 정책을 적용하여 쓰레드가 idle일 때 그 쓰레드를 keep alive timeout으로 종료할지의 여부. 체크박스로 정보를 표시하며, 체크가 되어 있으면 true, 그렇지 않다면 false를 의미한다. 기존 설정 값이 기본적으로 보여지며, 이 값을 동적으로 수정할 수 있다.

Current Size

해당 쓰레드 풀에서 현재 유지하고 있는 쓰레드의 갯수

Largest Size

해당 쓰레드 풀의 현재까지 쓰레드들을 관리하던 중 최대로 많이 유지했던 쓰레드의 갯수

Active

해당 쓰레드 풀에서 활성화된 쓰레드의 갯수

Task

해당 쓰레드 풀의 task 정보를 나타낸다. "Scheduled", "Completed" 두가지가 있으며, 전자의 경우 현재까지 스케줄링된 task의 갯수를 나타내며, 후자의 경우 스케줄링되어 현재 완료한 task의 갯수를 나타낸다.

Execution Time

해당 쓰레드 풀의 현재까지 실행된 쓰레드 수행시간(단위 milli-seconds). "Min", "Mean", "Max" 세가지가 있으며, 각각 최소의 수행시간, 평균 수행시간, 최대 수행시간을 나타낸다.

Queue Waiting Time

해당 쓰레드 풀의 현재까지 실행된 task의 대략적인 queueing 대기 시간(단위 milli-seconds). 해당 쓰레드 풀을 이용하여 task를 수행할 때, 수행을 스케줄링 한 후 실제 수행되기까지의 기다리는 시간을 의미한다. "Min", "Mean", "Max" 세가지가 있으며, 각각 최소의 대기시간, 평균 대기시간, 최대 대기시간을 나타낸다.

Work Queue

해당 쓰레드 풀에서 사용하고 있는 work queue의 정보. "Name", "Remaining Capacity", "Size"가 있으며 각각 해당 work queue의 이름, 허용 가능한 queue의 용량, 현재 queue가 가지고 있는 task의 갯수를 의미한다.

작업 쓰레드 정보

해당 쓰레드 풀에서 관리하고 있는 쓰레드들의 상세 정보를 표로 나타낸다. 우측 하단에 존재하는 "새로고침" 버튼을 클릭하면 실시간으로 변경된 쓰레드들의 정보를 업데이트 할 수 있다.

Name

해당 쓰레드를 식별하는 이름

Id

해당 쓰레드의 고유한 ID. java.lang.Thread에서 제공하는 ID를 사용한다.

Active

해당 쓰레드의 활성화 여부. 즉 해당 쓰레드가 task를 수행하고 있는지 아닌지를 알 수 있다.

Priority

해당 쓰레드의 우선순위(단위 Integer). 값이 "5"일 경우 normal, "1"일 경우 min, "10"일 경우 max의 priority를 의미한다. 이는 java.lang.Thread에서 제공하는 priority를 의미한다.

State

해당 쓰레드의 상태 정보. "NEW", "RUNNABLE", "BLOCKED", "WAITING", "TIMED_WAITING", "TERMINATED"가 있으며 java.lang.Thread에서 제공하는 Thread의 State를 이용한다. 해당 값의 자세한 의미는 java.lang.Thread의 javadoc을 참조하길 바란다.

Interrupted

해당 쓰레드가 Interrupted 상태인지에 대한 정보. java.lang.Thread#isInterrupted()와 같은 값이다.

Alive

해당 쓰레드가 물리적으로 alive 상태인지에 대한 정보. java.lang.Thread#isAlive()와 같은 값이다.

Daemon

해당 쓰레드의 종류가 Daemon 쓰레드인지에 대한 정보

WaitedTime

해당 쓰레드의 최근 기다린 시간(단위 milli-seconds). 보통 task가 없이 쓰레드가 idle상태일 때 기다리는 시간이 증가한다. "-1"일 경우 기다리는 시간을 계산할 수 없음을 의미한다.

RunTime

해당 쓰레드의 최근 수행 시간(단위 milli-seconds). "-1"일 경우 수행 시간을 계산할 수 없음을 의미한다(예를들어 waiting상태일 경우).

Interrupt

해당 쓰레드를 강제 종료 시키는 기능. Active상태일 경우만 Interrupt 버튼이 활성화 되며, 해당 쓰레드에 문제가 있을 시(예를들어 StackTrace에서 비정상적으로 Blocking된 쓰레드를 확인한 경우) 강제 종료 시킬 수 있는 기능이다.

StackTrace

해당 쓰레드의 현재의 StackTrace 정보. Active상태일 경우만 StackTrace를 제공하며, 해당 쓰레드의 "Name"에 해당하는 값을 클릭하면 정보를 얻을 수 있다.
