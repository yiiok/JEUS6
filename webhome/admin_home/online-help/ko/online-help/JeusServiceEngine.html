엔진

네비게이션 경로: Home > node-name > 엔진 컨테이너 > container-name > 엔진

개요

이 페이지에서는 이 엔진 컨테이너에 설정되어 있는 엔진 목록을 표의 형태로 볼 수 있다. 표에 나타난 각 컬럼을 분류하면 엔진의 정보(이름, 엔진 종류, 상태), 엔진의 삭제()이며, 표 아래에는 엔진을 등록할 수 있는 링크를 제공한다.

이름

엔진 컨테이너에 등록된 엔진의 이름. 각 엔진의 이름은 링크가 되어 있으며 이를 클릭하면 해당 엔진 설정 페이지로 이동한다. EJB 엔진의 경우 EJB 엔진의 기본 설정, 서블릿 엔진의 경우 서블릿 엔진의 1. 기본 설정, JMS 엔진의 경우 JMS 엔진의 기본 설정로 이동한다.

종류

이 엔진의 종류에 따라 ejb, web, jms, ws 중 한 가지 값이 나타난다.

  • ejb : EJB 엔진

  • web : 서블릿 엔진

  • jms : JMS 엔진

  • ws : 웹 서버 엔진

상태

이 엔진의 상태에 따라 Running, Stopped중 한 가지 값이 나타난다. 상태에 따라 아래에서 설명하는 명령 버튼들 활성/비활성화 된다.

  • Stopped : 엔진이 다운된 상태이다. 엔진 컨테이너 또는 노드가 Stopped 상태일 때 역시 엔진을 실행하지 못하므로 Stopped 상태이다.

  • Running : 엔진이 실행되고 있는 상태이다. 당연히 노드 및 엔진 컨테이너가 Running 상태이다.

삭제

Stopped 상태의 엔진만 휴지통 아이콘()이 보인다. 이를 클릭하면 삭제 확인 대화상자가 나타나고 확인 버튼을 클릭하면 엔진을 엔진 컨테이너에서 제거한다. 엔진 리스트에 엔진이 하나 밖에 없다면 엔진 컨테이너에는 적어도 하나의 엔진이 존재 해야 하므로 그 엔진을 삭제할 수 없다.

새 xxx 엔진 생성

엔진 컨테이너는 하위에 엔진 종류별로 하나의 엔진만을 가질 수 있다. 따라서 이미 등록되어 있는 종류의 엔진을 제외한 나머지 종류의 엔진만을 생성할 수 있는 버튼들이 표 아래에 보인다. 이 버튼을 클릭하면 새 엔진 생성 페이지로 이동한다. 자세한 내용은 새 엔진 생성 페이지의 도움말을 참조하고 생성을 하고 나면 엔진 목록에 새로운 엔진이 Stopped 상태로 추가된다.

새 엔진 생성

네비게이션 경로: Home > node-name > 엔진 컨테이너 > container-name > 엔진 > 새 엔진 생성

엔진은 상위 엔진 컨테이너에 종류별로 하나씩만 등록할 수 있다. 따라서 새 엔진을 등록할 경우에는 엔진 컨테이너에 등록되어 있지 않은 종류의 엔진만을 등록할 수 있다. 또한 각 엔진별로 설정의 차이가 있으므로, 새 엔진 생성 페이지는 엔진의 종류별로 각각 존재한다.

새 EJB 엔진 생성

Type

EJB. EJB엔진 생성 페이지이므로 항상 이 값으로 고정된다.

Name

생성하고자 하는 EJB 엔진의 이름. 하나의 노드 안에서는 같은 이름을 가진 EJB 엔진이 있어서는 안된다.

JEUSMain.xml에 <engine-command>의 <name>값으로 설정되며, JEUS_HOME/config/${node-name}/${node_name}_ejb_${name} 아래에 EJBMain.xml이 생성된다.

새 서블릿 엔진 생성

서블릿 엔진을 부팅시킬 때 필요한 최소한의 값을 입력받아, 생성 시에 WEBMain.xml을 만든다. 입력하지 않은 다른 값들은 기본값으로 설정된다.

Type

SERVLET. 서블릿 엔진 생성 페이지이므로 항상 이 값으로 고정된다.

Name

생성하고자 하는 서블릿 엔진의 이름. 하나의 노드 안에서는 같은 이름을 가진 서블릿 엔진이 있어서는 안된다.

JEUSMain.xml에 <engine-command>의 <name>값으로 설정되며, JEUS_HOME/config/${node-name}/${node_name}_servlet_${name} 아래에 WEBMain.xml이 생성된다.

리스너 아이디

기본적인 http 리스너의 아이디. 모든 서블릿 엔진 내에서 유일한 아이디 값을 가져야 한다.

리스너 포트

기본적인 http 리스너가 사용할 포트. 모든 서블릿 엔진 내에서 유일한 포트 값을 가져야 한다.

새 JMS 엔진 생성

Type

JMS. JMS 엔진 생성 페이지이므로 항상 이 값으로 고정된다.

Name

생성하고자 하는 JMS 엔진의 이름. 하나의 노드 안에서는 같은 이름을 가진 JMS 엔진이 있어서는 안된다.

JEUSMain.xml에 <engine-command>의 <name>값으로 설정되며, JEUS_HOME/config/${node-name}/${node_name}_jms_${name} 아래에 JMSMain.xml이 생성된다.

서비스 채널 이름

서비스 채널의 이름이다. Connection Factory 에 채널 정보를 지정하기 위해 용도로 사용된다.

서비스 채널 포트

해당 서비스 채널의 서비스 포트 번호이다.
