개요

이 화면에서는 현재 노드에 생성된 클러스터 데이터 소스 정보를 보여준다. 각각의 클러스터 데이터 소스 정보는 다음의 항목으로 구성되어 있다.

  • 데이터 소스 ID

    클러스터 데이터 소스를 구별하는 ID. 새 JDBC 클러스터 데이터 소스 생성시 클러스터 데이터 소스 ID를 입력하지 않았을 경우에는 익스포트 이름이 클러스터 데이터 소스 ID가 된다.

  • 커넥션 준비

    클러스터 데이터 소스로 묶인 컴포넌트 데이터 소스들을 실제 사용하기 전, 각각에 대하여 미리 커넥션 풀을 구성할지 여부. 이 값이 true로 설정이 되면 커넥션 풀을 미리 구성하므로 부하가 있을 수 있으나 최초 커넥션 요청에 대한 응답을 빠르게 할 수 있다.

  • 데이터 소스 수

    클러스터 데이터 소스로 묶인 컴포넌트 데이터 소스 개수

  • 명령

    클러스터 데이터 소스의 익스포트 이름으로 JNDI에 등록한다.

새 클러스터 데이터 소스 생성

클러스터 데이터 소스 화면 아래의 '새 JDBC 클러스터 데이터 소스 생성' 링크를 클릭하여 설정 화면으로 넘어간다. 그 이후에는 다음과 같은 순서로 설정을 한다.

  • 기본 설정

    이 화면에서는 클러스터 데이터 소스에 대한 기본 설정을 한다. 다음 버튼을 클릭하면 '데이터 소스 이름' 화면으로 넘어간다.

    • data_source_idField : 노드에서 관리하는 클러스터 데이터 소스 ID. 설정되지 않으면 'export_nameField'의 설정이 이것을 대신한다.

    • export_nameField : JNDI에 등록되는 이름

    • data_source_targetField : 클러스터 데이터 소스를 사용할 수 있는 엔진 컨테이너 이름. 설정하지 않으면 노드에 속한 모든 엔진 컨테이너에서 사용할 수 있게 된다.

    • load_balanceField : load balancing을 사용할 지 여부. 이 설정 값이 true이면 'is_pre_connField'와 'use_failbackField' 설정은 무시된다.

    • is_pre_connField : 클러스터 데이터 소스로 묶인 컴포넌트 데이터 소스들을 실제 사용하기 전, 각각에 대하여 미리 커넥션 풀을 구성할지 여부. 이 값이 true로 설정이 되면 커넥션 풀을 미리 구성하므로 부하가 있을 수 있으나 최초 커넥션 요청에 대한 응답을 빠르게 할 수 있다.

    • use_failbackFiled : back-up 데이터 소스로의 fail-over 이후 primary 데이터 소스로의 fail-back을 수행하는 기능을 사용할지 여부. fail-back이 수행되기 위해서는 primary 데이터 소스에 대한 check-query 및 check-query-period가 반드시 설정되어 있어야 한다.

  • 데이터 소스 이름

    이 화면에서는 클러스터 데이터 소스로 묶을 컴포넌트 데이터 소스를 지정한다. 지정 순서에 따라 사용순서가 결정 된다. 컴포넌트 데이터 소스를 SELECT / DESELECT로 추가 / 제거할 수 있고 그것들의 순서를 UP / DOWN으로 변경할 수 있다. 다음 버튼을 클릭하면 '생성' 화면으로 넘어간다.

  • 생성

    생성 버튼을 클릭하여 데이터 소스를 생성한다.

클러스터 JDBC 데이터 소스 서비스

처음 화면에서 해당 클러스터 데이터 소스 ID를 클릭하면 클러스터 데이터 소스의 설정을 볼 수 있다. 이 페이지에서는 처음 데이터 소스를 생성할 때 지정했던 값들을 출력해주며 제한적으로 설정을 변경할 수 있다. 설정을 변경한 후에는 확인 버튼을 눌러 변경 내용을 반영하도록 한다. 각 설정에 대한 설명은 위의 '새 클러스터 데이터 소스 생성'을 참조하기 바란다.
