JEUS 매니저

네비게이션 경로: Home > node-name

개요

이 페이지에서는 JEUS의 전반적인 기능들에 대한 설정 및 제어 기능을 제공한다.

JEUS 매니저는 JEUS의 하위 시스템을 광범위하게 관리하기 위한 하나의 어플리케이션이다. 보통 노드와 JEUS 매니저는 같은 것은 아니지만 매우 밀접하게 관련되어 있다. 각 JEUS 매니저는 반드시 한 개의 노드를 관리하고, 각 노드는 반드시 한 개의 JEUS 매니저에 의해서 관리된다. 이런 이유로 인해 JEUS WebAdmin에서의 JEUS 매니저 관리는 노드 관리까지 포함하고 있다.

JEUS 매니저 - 기본 설정

노드의 가장 기본적인 것들에 대한 설정을 한다.

Name

JEUS 시스템 내에서 노드의 식별자로 클러스터링 환경 내에서 유일한 이름을 가진다. 기본적으로 노드의 이름은 노드와 노드 관리자가 동작하고 있는 물리적인 호스트의 네트워크 식별 이름과 동일한 이름이다. 주의할 것은 노드의 이름은 영문자와 숫자의 조합으로 이루어지되, 영문자로 시작하는 문자열이어야 한다.

고급 선택 사항
Backup Node

현재 노드가 어떤 노드에 심각한 장애가 발생할 경우, 이 노드를 대신해서 동작해야 할 대상 노드의 이름이다.

Auto Restart

엔진 컨테이너가 비정상 종료 되었을 때 매니저에서 컨테이너를 자동으로 재기동 해 줄지 여부를 결정한다. 각 엔진 컨테이너에 한 설정이 우선시 되고 엔진 컨테이너에 해당 설정이 없으면 매니저의 설정을 따른다. 엔진 컨테이너와 매니저 모두 이 설정이 없다면 기본 값은 true로 매니저에서 컨테이너를 재기동 해준다.

Sequential Start

엔진 컨테이너들이 순차적으로 시작될 지 여부를 결정하는 설정이다. 만약 false로 설정될 경우 각 엔진 컨테이너가 동시에 시작된다. 기본값은 false 이다.

Log Stdout To Raw Format

엔진 컨테이너 로그에 출력되는 표준출력/에러의 출력방식에 대해 설정한다. 기본적으로 엔진 컨테이너에 한 설정이 최우선시 되고 이 설정이 없다면 매니저의 설정을 따른다. 엔진 컨테이너와 매니저 모두에 이 설정이 없다면 기본 값은 true이다.

JEUS 매니저 - 리스너

리스너는 서브 컴포넌트 간에 혹은 클러스터링 환경에서 다른 JEUS와 통신하기 위한 소켓 포트이다. 이 페이지에서는 이 소켓 리스너 및 소켓 커넥션 요청 처리에 필요한 각 속성값들을 설정한다.

기본 설정

Backlog

JEUS system listener port인 JEUS Base Port에 대한 backlog값을 지정한다.

Read Timeout

JEUS Baseport로 요청이 들어왔을 때 각 요청에 맞는 서비스로 분배하기 위해 패킷을 읽을 때의 읽기함수가 블록킹되는 시간을 설정한다. 기본값은 30초이고 단위는 ms이다.

Listen Service Ip Only

머신에 여러개의 IP가 할당되어 있을 때 여러개의 IP 중에서 특정 IP address (jeus.net.localip)로만 서비스 IP를 노출할지 여부를 지정한다. 이 값을 true로 설 정하면 매니저와 엔진 컨테이너의 baseport와 웹어드민 포트, RMI 포트, JMS 포트, HTTP 리스너 포트 등은 모두 jeus.net.localip 로 지정된 서비스 IP로만 바인딩하게 된다. 기본값은 false로 머신에 할당된 어떤 IP로든 서비스될 수 있다.

SSL

Jeus Security system에 관련된 SSL 속성을 지정한다. 이 값을 설정하면 Jeus Security system을 사용하는 모든 소켓 커넥션에 SSL이 적용된다.

Port

Jeus Security system이 SSL connection에 사용할 listen port를 지정한다.

쓰레드 풀

Jeus system listener port(JEUS_BASEPORT)에 요청되는 소켓 커넥션 처리를 위한 쓰레드 풀을 설정한다.

Min

pooling되는 객체의 최소값을 지정한다.

Max

pooling되는 객체의 최대값을 지정한다.

Period

pooling되는 객체를 정리하는 시간을 지정한다.

JEUS 매니저 - 서비스

Class Ftp

EJB 스텁을 클라이언트로 전송하기 위한 Class FTP 서버를 사용할 지의 여부를 결정하는 설정이다. 만약 false로 설정해 놓고 EJB 클라이언트에 EJB 스텁이 없다면 에러가 발생할 수 있다. 기본값은 false이다.

JEUS 매니저 - WebAdmin

WebAdmin에 대한 설정을 한다.

WebAdmin 설정
Enable Webadmin

JEUS에서 웹 기반의 관리 툴인 WebAdmin을 사용할지를 결정한다. 기본값은 true이다.

Use Baseport

JEUS BASEPORT(9736)을 통해서 webadmin을 접속할지 여부를 결정한다. false인 경우에는 JEUS BASEPORT + 8(9744)로 접속해야한다.

Allowed Server

WebAdmin에 접근할 권한을 갖는 호스트 리스트의 IP 주소를 설정한다. 여기에 설정된 주소의 호스트만이 WebAdmin에 접근할 수 있다. 설정되는 IP 주소에 wildcard(*)를 사용할 수 있다. (예: 192.168.*.*)

plain listener

http request를 받는 WebAdmin listener에 대한 설정을 한다.

enabled

이 listener를 사용할 지 안할 지 여부를 설정한다.

port

port number를 직접 지정하고 싶을 때 사용. default는 use-baseport가 true일 경우 baseport이고 그 외에는 baseport+8 이다.

ssl listener

https request를 받는 WebAdmin listener에 대한 설정을 한다.

enabled

이 listener를 사용할 지 안할 지 여부를 설정한다.

port

port number를 직접 지정하고 싶을 때 사용. default는 baseport+9이다.

고급 선택 사항
Client Auth

client의 인증 여부를 설정한다. true이면 server가 client에게 인증서를 요청하여 client에 대한 인증을 수행하고, false이면 client에 대한 인증과정을 수행하지 않는다. 일반적으로 B2B를 제외한 보통의 경우에는 client 인증을 수행하지 않는다. 기본 값은 false 이다.

SSL Protocol

암호화/복호화하는데 사용되는 SSL 프로토콜을 지정한다. dafault value가 Sun JVM을 사용할 때는 TLS, IBM JVM을 사용할 때는 SSL 이다.

Cipher Suite

SSL handshaking후에 실제 데이타를 보낼 때 사용하는 암호슈트들을 지정한다. 일반적으로 JDK에서 제공하는 cipher-suite를 사용하며, default로 제공되지 않는 cipher suite를 사용할 때 사용한다.

Keystore File

key와 인증서를 저장하고 있는 파일을 지정한다. 절대경로, 상대경로 모두 허용된다. 만일 상대경로가 사용된다면 jeus_home/config/nodename 아래에서 해당 파일을 찾는다. 그리고 JVM인자로 jeus.ssl.keystore과 javax.net.ssl.keyStore을 이용해서 설정할 수 있다. 우선순위는 설정 값 > jeus > javax 이다. 기본 값은 jeus_home/config/nodename/keystore 이다.

Keystore Pass

keystore파일을 열기 위한 password이다. 이 설정은 JVM인자로 jeus.ssl.keypass과 javax.net.ssl.keyStorePassword를 이용해서 설정할 수 있다. 우선순위는 설정 값 > jeus > javax 이다. 기본 값은 jeuskeypass 이다.

Keystore keypassword

keystore 파일의 key를 열기 위한 password이다. 모든 key들은 같은 암호로 저장되어 있어야 한다. 설정되지 않으면 keystore 암호와 동일한 암호를 사용한다.

Keystore Type

keystore의 타입을 지정한다. Sun의 keytool에 의해서 keystore를 생성한다면 JKS(Java's Key Store)를, OpenSSL이나 Microsoft KeyManager로 keystore를 생성한다면 PKCS12를 사용해야 된다. 이 설정은 JVM인자로 javax.net.ssl.keyStoreType을 이용해서 설정할 수 있다. 우선 순위는 설정 값 > javax 이다. 기본 값은 JKS 이다.

Key Management Algorithm

keystore에 저장할 key에 대한 관리 알고리즘을 설정한다. 기본적으로 Sun JVM을 사용할 경우 SunX509가 지정되며, IBM JVM을 사용할 경우에는 IbmX509가 지정된다. 이 설정은 JVM인자로 ssl.KeyManagerFactory.algorithm을 이용해서 설정할 수 있다. 우선 순위는 설정 값 > javax 이다.

Key Alias

SSL을 사용할때 이용되는 서버 인증서의 키 Alias이름을 설정한다.

Truststore File

client 인증서를 저장하고 있는 파일을 지정한다. 절대경로, 상대경로 모두 허용된다. 만일 상대경로가 사용된다면 jeus_home/config/nodename 아래에서 해당 파일을 찾는다. 그리고 JVM인자로 jeus.ssl.truststore과 javax.net.ssl.trustStore을 이용해서 설정할 수 있다. 우선순위는 설정 값 > jeus > javax 이다. 기본 값은 jeus_home/config/nodename/truststore 이다.

Truststore Pass

truststore파일을 열기 위한 password이다. 이 설정은 JVM인자로 jeus.ssl.trustpass과 javax.net.ssl.trustStorePassword를 이용해서 설정할 수 있다. 우선순위는 설정 값 > jeus > javax 이다. 기본 값은 jeustrustpass 이다.

Truststore Type

truststore의 타입을 지정한다. Sun의 keytool에 의해서 keystore를 생성한다면 JKS(Java's Key Store)를, OpenSSL이나 Microsoft KeyManager로 keystore를 생성한다면 PKCS12를 사용해야 된다. 이 설정은 JVM인자로 javax.net.ssl.trustStoreType을 이용해서 설정할 수 있다. 우선 순위는 설정 값 > javax 이다. 기본 값은 JKS 이다.

Trust Management Algorithm

truststore에 저장할 trust에 대한 관리 알고리즘을 설정한다. Sun JVM을 사용할 경우 SunX509가 지정되며, IBM JVM을 사용할 경우에는 IbmX509가 지정된다. 이 설정은 JVM인자로 ssl.TrustManagerFactory.algorithm을 이용해서 설정할 수 있다. 우선 순위는 설정 값 > javax 이다.

crl File

Certificate Revocation Lists를 저장하고 있는 파일을 지정한다. 절대경로, 상대경로 모두 허용된다. 만일 상대경로가 사용된다면 jeus_home/config/nodename 아래에서 해당 파일을 찾는다.

JEUS 매니저 - 에러 로그

JEUS 매니저에서 사용할 로거를 설정한다.

이 페이지에서는 JEUS 매니저에 설정된 로거의 목록을 보여주고, 등록된 로거의 설정을 변경/삭제하거나 새로운 로거를 추가할 수 있다.

로거 테이블

이 테이블에서는 등록된 로거의 이름과 Level을 볼 수 있으며, 등록된 로거를 삭제할 수도 있다.

이름

등록된 로거의 이름을 나타내며, 이 로거의 링크를 클릭하면 해당 로거의 설정을 변경할 수 있는 페이지(로깅 - 설정)로 이동한다.

LEVEL

로거의 로깅 레벨을 나타낸다. 로깅 레벨은 아래와 같이 12개가 있다.

레 벨설 명
FATALJEUS 4.x대의 logger의 level로 compatibility를 위해 지원한다.SEVERE에 해당하는 level이다.
NOTICEWARNING에 해당하는 level이다.
INFORMATIONINFO에 해당하는 level이다.
DEBUGFINE에 해당하는 level이다.
SEVERE치명적인 실패, 오류(serious failure)에 대한 로깅 레벨
WARNING잠재적인 문제(potential problem)에 대한 로깅 레벨
INFO안내 메세지(information message)에 대한 로깅 레벨
CONFIG설정 메세지(static configuration message)에 대한 로깅 레벨
FINE트레이스(trace) 정보에 대한 로깅 레벨
FINERFINE 레벨보다 꽤 세세한 트레이스(trace) 정보에 대한 로깅 레벨
FINEST가장 세세한 트레이스(trace) 정보에 대한 로깅 레벨
ALL모든 메세지에 대한 로깅 레벨

참고

각 레벨에 대한 자세한 설명은 J2SE Logging API의 Level 클래스나 Javadoc을 참고하기 바란다.

삭제()

표의 맨 마지막 컬럼 나타난 을 누르면 해당 로거를 삭제할 것인지 확인 메세지가 출력되고, '확인'을 누르면 삭제된다.

로거 추가

표 아래에 ' 로거 추가'를 클릭하면 로깅 - 로거 추가로 이동하여 새로운 로거를 등록할 수 있다.

JEUS 매니저 - 보안

JEUS Security에 대한 설정이다.

기본 설정

Use Nio

JEUS Security 에서 Network 통신 방식에 NIO방식 사용 여부를 정의한다.

Connect Retries

JEUS Security NetworkService에서 connection retry count값을 정의한다. 기본값은 10번이다.

도메인 설정

JEUS 부트 시에 Security를 설정할 도메인을 정의한다.

현재 설정되어 있는 도메인이 표에 나타나다. 해당 도메인 이름을 클릭하면 도메인 설정을 변경(보안 도메인 설정)할 수 있으며, 표 아래 ' 새 보안 도메인 생성' 을 클릭하면 보안 생성 페이지(새 보안 도메인 생성)로 이동하여 새 도메인을 추가할 수 있다.

JEUS 매니저 - 제어

클러스터 노드를 포함한 모든 노드를 제어(부트, 다운, 종료)할 수 있다.

부트

부트는 현재 노드 및 클러스터 상에 있는 노드의 모든 컨테이너를 시작시키는 명령이다. 하나의 노드에 대해서 엔진 컨테이너 모두 시작을 하는 것과 동일하게 동작한다. 가능한 노드 목록에는 현재 READY 상태의 노드들을 보여준다.

각각의 노드를 선택하거나 모든 노드를 한꺼번에 선택할 수 있다. [확인] 버튼을 누르면 부트가 실행되며, [취소] 버튼을 누르면 선택 사항이 모두 해제된다.

부트가 완료되어 컨테이너가 정상적으로 시작된 경우에는 JEUS 노드 트리 뷰에 있는 엔진 컨테이너 하위의 컨테이너 아이콘이 처럼 된다.

다운

다운은 현재 노드 및 클러스터 상에 있는 노드의 모든 컨테이너를 다운시키는 명령이다. 하나의 노드에 대해서 엔진 컨테이너 모두 다운을 하는 것과 동일하게 동작한다. 현재 노드 및 클러스터 상에 있는 노드의 기동되어 있는 컨테이너를 다운시킨다. 가능한 노드 목록에는 현재 READY 상태의 노드들을 보여준다.

각각의 노드를 선택하거나 모든 노드를 한꺼번에 선택할 수 있다. [확인] 버튼을 누르면 다운이 실행되며, [취소] 버튼을 누르면 선택 사항이 모두 해제된다.

다운이 완료되어 컨테이너가 성공적으로 다운된 경우에는 JEUS 노드 트리 뷰에 있는 엔진 컨테이너 하위의 컨테이너 아이콘이 처럼 된다.

종료

현재 노드 및 클러스터 상에 있는 노드들의 JEUS 매니저를 종료한다.

각각의 노드를 선택하거나 모든 노드를 한꺼번에 선택할 수 있다. [확인] 버튼을 누르면 종료가 실행되며, [취소] 버튼을 누르면 선택 사항이 모두 해제된다.

JEUS 매니저 - 클러스터

일반적으로 시스템 과부하를 신뢰성 있게 처리하기 위해 여러 JEUS를 연결해서 하나의 클러스터링을 구성한다.

클러스터링을 구성하는 방법에는 정적인 방법과 동적인 방법이 존재한다.

정적으로 클러스터링을 구성한다는 것은 JEUSMain.xml에 클러스터링에 참여하는 노드들을 미리 등록하고 부팅시에 클러스터링을 구성하는 것을 말한다. JEUS WebAdmin을 통해서 정적 클러스터링을 구성하기 위해서는 다음의 조건을 만족해야 한다.

  • 클러스터링에 참여하고자 하는 노드들의 JEUS 매니저가 동작 중이어야 한다.

  • 각 노드들은 시스템의 hosts 파일에 등록되어 있거나 JEUS의 가상 호스트 설정 파일(JEUS_HOME/config/vhost.properties)에 등록되어 있어야 한다.

  • 가상 호스트가 아닌 실제 물리적인 노드를 사용할 경우에는 JEUS_BASEPORT가 같아야 한다.

  • 가상 호스트는 가상 호스트끼리 물리적 노드는 물리적 노드끼리만 클러스터링이 가능하다. 가상 호스트 설정은 가상 호스트를 참조하기 바란다.

가능한 노드 목록에는 현재 노드와 클러스터링으로 연결된 노드들이 출력된다. 또한 JEUS 클러스터 뷰에도 각 노드가 표시된다.

추가

클러스터링에 추가하고자 하는 노드의 이름을 입력하고 추가 버튼을 누른다. 네트워크 상에서 해당 노드 이름으로 실행 중인 JEUS 노드를 검색하여 클러스터에 추가한다. 해당 노드 이름이 존재하지 않는 경우에는 "host_name(은)는 알려지지 않은 호스트입니다. 노드 이름을 확인해 주십시오."라는 메세지가 출력된다. 정상적으로 추가된 경우에는 JEUS 클러스터 뷰와 이 페이지의 가능한 노드 목록에 노드 이름이 추가된다.

삭제

JEUS의 클러스터링에서 노드를 삭제하는 것은 단순히 JEUSMain.xml 의 노드 목록에서 삭제하는 것을 말한다. 곧 운영중인 클러스터링에서의 동적 삭제는 불가능하며 노드를 삭제하고 전체 시스템을 리부팅 할 경우에만 클러스터링에서 삭제가 완전히 되므로 주의해야 한다.

삭제하고자 하는 노드를 가능한 노드 목록에서 선택한 후에 삭제 버튼을 누르면 된다. 삭제가 성공적으로 이루어졌다면, JEUS 클러스터 뷰와 이 페이지의 가능한 노드 목록에서 해당 노드 이름이 삭제된다.

JEUS 매니저 - 배포

현재 선택된 노드의 설정(JEUSMain.xml)을 전체 노드에 동일하게 복사하는 것을 말한다. 이 기능은 노드들을 부팅하기 전에 배포를 통해서 동일한 설정을 적용 할 경우에만 사용하는 것으로 노드들이 이미 부팅되었을 경우에는 사용하지 않는 것이 좋다.

가능한 노드 목록에는 클러스터 상에 모든 노드들이 나타나며, 각각의 노드 또는 노드 전체를 선택할 수 있다. 배포하기 위한 노드가 없는 경우에는 "가능한 노드가 없습니다."라는 메세지가 출력된다.
