MQ 데이터 소스

개요

JEUS는 JEUS 매니저 서비스에 MQ 데이터 소스를 설정함으로써 타사 MQ 제품과 연동이 가능하다. 이 페이지는 현재 사용 가능한 MQ 데이터 소스의 목록을 보여주고, 특정 데이터 소스의 설정을 변경하거나 새 MQ 데이터 소스를 추가하는 기능을 제공한다.

MQ 데이터 소스 목록에는 다음 항목들이 표시된다.

익스포트 이름

해당 데이터 소스의 JNDI 이름. 클릭하면 MQ 데이터 소스 설정 페이지로 이동한다.

벤더

이 MQ 데이터 소스 서비스를 제공하는 벤더 이름. ibmmq, sonicmq, others 중 한 가지 값이 표시된다.

팩토리 클래스

이 데이터 소스 서비스에 해당하는 팩토리 클래스 이름.

리소스 타입

데이터 소스의 JMS 리소스 타입. QCF, TCF, Q, T, XAQCF, XATCF, LOCALXAQCF, LOCALXATCF 중 하나의 값을 갖는다.

명령

바인드 버튼을 누르면 해당 MQ 데이터 소스를 JNDI 서버에 등록할 수 있다. 이미 등록되어 있으면 이 버튼은 비활성화 된다.

삭제

휴지통 아이콘()을 클릭하면 삭제 확인 대화상자가 나타나고, 확인 버튼을 누르면 해당 데이터 소스를 JEUS 매니저에서 제거할 수 있다.

표 우측 하단에 있는 새 MQ 데이터 소스 생성 링크를 클릭하면 새 MQ 데이터 소스 생성 페이지로 이동한다.

새 MQ 데이터 소스 생성

새 MQ 데이터 소스를 생성하는 과정은 1. 데이터 소스, 2. 속성, 3. 생성 세 단계로 구성된다.

1. 데이터 소스

이 단계에서는 데이터 소스로 연결할 타사 MQ 서비스의 벤더와 리소스 타입을 선택하게 된다. 벤더별 리소스 타입은 아래 표와 같다.

IBMSONICOTHERS

IBM - QCF Type

IBM - TCF Type

IBM - Q Type

IBM - T Type

IBM - XAQCF Type

IBM - XATCF Type

IBM - LOCALXAQCF Type

IBM - LOCALXATCF Type

SONIC - QCF Type

SONIC - TCF Type

SONIC - Q Type

SONIC - T Type

SONIC - XAQCF Type

SONIC - XATCF Type

SONIC - LOCALXAQCF Type

SONIC - LOCALXATCF Type

OTHERS - XAQCF Type

OTHERS - XATCF Type

2. 속성

다음은 데이터 소스의 속성을 지정하는 단계이다. 앞 단계에서 선택한 벤더 및 리소스 타입에 따라 설정할 수 있는 필드가 다르다.

Vendor

이전 단계에서 선택한 벤더가 표시된다.

Factory Class Name

IBM이나 SONIC을 벤더로 선택한 경우에는 해당 리소스 타입의 팩토리 클래스 이름이 표시되지만, 그렇지 않은 경우에는 팩토리 클래스 이름을 직접 입력할 수 있도록 되어 있다. 해당 제품의 매뉴얼을 참고하여 입력하도록 한다.

Resource Type

이전 단계에서 선택한 리소스 타입이 표시된다.

Export Name

이 데이터 소스의 JNDI 이름을 입력한다.

추가 속성들

팩토리 클래스가 제공하는 속성이 있으면 이 영역에 <name>-<value>의 쌍을 속성별로 한 줄에 입력하도록 한다. Factory Class Name 필드에서 지정한 팩토리 클래스의 API를 참고하기 바란다.

그 밖에 벤더 및 리소스 타입에 따라 추가로 설정 가능한 항목이 있는 경우가 있는데, 해당 제품 매뉴얼을 참고하여 설정할 수 있다.

3. 생성

이 MQ 데이터 소스를 지금 바인드 하시겠습니까? 항목에 체크하면 생성과 동시에 새 MQ 데이터 소스를 JNDI 서버에 등록한다.

생성 버튼을 클릭하면 지금까지 설정한 대로 새로운 MQ 데이터 소스가 JEUS 매니저에 추가되면서 개요 페이지로 이동한다.

MQ 데이터 소스 설정

이 페이지에서는 해당 MQ 데이터 소스의 설정을 확인하고 변경할 수 있다. 하지만 다음 기본 속성은 변경할 수 없다.

  • Vendor

  • Factory Class Name

  • Resource Type

각 항목의 의미는 새 MQ 데이터 소스 생성의 2. 속성 단계에 설명되어 있다.
