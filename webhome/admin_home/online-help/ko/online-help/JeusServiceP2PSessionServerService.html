설정

기본 설정

분산 세션 서버의 기본 설정을 정의한다. 분산 세션 서버를 사용하기 위한 필수 항목(Required Field)은 없다.

Connect Timeout

분산 세션 서버간의 Connection 타임 아웃을 설정한다.

Read Timeout

분산 세션 서버간의 응답 메세지의 타임 아웃을 설정한다. 만약 부하가 많이 걸리고 네트워크의 상황이 좋지 않아 메세지 전송이 원활하지 못할 경우 이 타임아웃 값을 늘려주도록 한다.

Backup Trigger

백업 분산 세션 서버로 업데이트 대상의 세션들을 백업할 때 과도한 네트워크의 부하를 줄이기 위해 특정 개수의 세션이 모일 때까지 버퍼링 하여 한번에 전송하도록 한다. 이때 최대 몇 개의 세션들을 버퍼링 할지의 개수를 설정한다.

Check To

지정된 Check 타임아웃 주기마다 업데이트 대상의 세션들을 백업 분산 세션 서버로 백업한다. 이 주기 값이 짧을 수록 현 분산 세션 서버와 백업 분산 세션 서버와의 세션 동기화가 잘 이루어지지만 업데이트 대상의 세션들이 자주 발생할 경우 네트워크에 부하가 걸릴 수 있다.

Check Level

업데이트 대상의 세션들을 정의하기 위한 레벨을 선택한다. 보통 set으로도 충분 하나, 항상 모든 세션을 백업 분산 세션 서버와 실시간으로 동기화시키고 싶을 경우 all을 사용하도록 한다.

쓰레드 풀

분산 세션 서버는 여러 서블릿의 세션 컨테이너들과 복잡한 접속을 맺어 많은 세션들을 분산 저장, 관리할 수 있다. 이때 많은 세션, Connection들이 어느 한쪽으로 집중될 수 있기 때문에 특별히 쓰레드들을 풀링하여 관리한다. 동시 Connection의 상황, 세션 데이타의 전송 양, 크기에 따라 요청 처리 쓰레드 풀 값을 조절하도록 한다.

Min

최소의 쓰레드 풀링 개수를 설정한다.

Max

최대의 쓰레드 풀링 개수를 설정한다.

Period

풀링 쓰레드가 정리되는 주기를 설정한다.

파일 스토리지

분산 세션 서버는 여러 서블릿의 세션 컨테이너들과 복잡한 접속을 맺어 많은 세션들을 메모리에 분산 저장, 관리할 수 있다. 이때 많은 세션들이 어느 한쪽으로 세션 서버로 집중될 경우 서블릿 엔진에 과도한 메모리 부하가 걸릴 수 있다. 이를 방지하기 위해 Passivation 정책을 통해 메모리 세션들을 비활성화 상태로 바꾸어 파일 스토리지에 저장한다. 파일 스토리지를 사용할 경우 세부 항목들을 설정하기 위한 값들이다.

참고

여기서 파일 스토리지 설정을 할 경우 하위 세션 라우터들에서 공통적으로 이 설정값을 사용하게 된다. 하위 세션 라우터에서 특별한 파일 스토리지를 설정하고자 할 경우 설정을 참고 하도록 한다.

Startup Clear To

이 설정한 값보다 파일 스토리지의 Access시간이 더 오래되었을 경우 유효하지 않은 파일 스토리지로 간주하여 파일 스토리지를 초기화 한다. 오래된 파일 스토리지로 인해 디스크의 용량이 불필요하게 늘어나는 것을 방지하기 위함이다.

Path

파일 스토리지의 절대 경로를 지정한다. 이 설정은 파일 스토리지 이름을 포함한 절대 경로로 지정되어야 한다.

Passivation To

이 Passivation 타임아웃을 이용하여 특정 주기 동안 사용되지 않는 세션에 대해 파일 스토리지로 세션을 옮기는 작업을 수행하여야 한다. Passivation 작업을 통해 불필요하게 메모리에 오래 저장되어 있는 세션들을 임시로 파일 스토리지로 옮기어 비 활성화 시킨다.

Min Hole

설정된 값 이상의 파일 IO가 일어날 경우 파일 스토리지를 재 정리하여 파일 스토리지의 크기를 최적화 시킨다.

Packing Rate

먼저 Packing Rate을 계산한다. Packing Rate이란 (파일 IO횟수 / 현 세션의 수)를 의미하며 소수점을 가진 값이 된다. 설정된 값의 비율이 이 설정한 Packing Rate 이상이 되었을 경우 파일 스토리지를 재 정리하여 파일 스토리지의 크기를 최적화 시킨다.

세션 라우터

이 페이지에서는 특정 엔진에서 사용하고 있는 분산 세션 서버의 설정 목록을 표의 형태로 볼 수 있다. 표에는 분산 세션 서버 종류 별로 다음 정보가 표시된다.

Engine Name

설정된 분산 세션 서버의 엔진 이름 정보가 나타난다. 보통 EJB 엔진에서 사용할 경우 ejb-engine-name , 서블릿 엔진에서 사용할 경우 servlet-engine-name이 나타나며, 이는 해당 엔진 당 고유한 분산 세션 서버 이름을 의미한다. 해당 이름을 클릭하면 설정 페이지로 이동한다.

Replication Group

설정된 분산 세션 서버의 논리적인 그룹 정보가 나타난다. 이 지정된 이름은 다른 분산 세션 서버 설정에서 Backup Group의 설정에 필요하다. 즉, 다른 분산 세션 서버에서 해당 세션 서버를 되도록이면 백업 분산 세션 서버로 사용하고자 할때 Backup Group에 해당 설정값을 적어주면 된다.

Backup Group

설정된 분산 세션 서버의 특별히 선호하는 백업 분산 세션 그룹 정보가 나타난다. 다른 분산 세션 서버 설정 중의 Replication Group 값이 나타날 수 있다. 이 값이 설정되어 나타났을 경우 이 값의 그룹 이름을 백업 분산 세션 서버로 선택하는데 최우선 조건으로 사용한다.

Location

설정된 분산 세션 서버의 물리적인 Machine 정보, 즉 위치(Location) 정보가 나타난다. 하나의 서버 이름일 수도 있고 여러 서버가 rack안에 구성된 경우 rack의 상징적인 이름일 수도 있다. 내부적으로 백업 분산 세션 서버를 구성할 때 여기서 설정된 Location 정보와 되도록이면 다른 Location에 존재하는 분산 세션 서버를 백업으로 선택할 것이다.

삭제

분산 세션 서버가 설정되어 나타난 경우 휴지통 아이콘()이 보인다. 이를 클릭하면 삭제 확인 대화상자가 나타나고 확인 버튼을 클릭하면 해당 분산 세션 서버 설정이 제거한다.

또한 표 아래의 버튼으로 새로운 분산 세션 서버 설정을 추가 할 수 있다.

세션 라우터 생성

분산 세션 서버 설정 정보 표 아래에 "세션 라우터 생성"을 클릭하면 생성 페이지로 이동한다. 자세한 내용은 생성 페이지의 도움말을 참조한다. 생성을 하고 나면 표 목록에 새로운 분산 세션 서버 설정이 하나 추가된다.
