새 서비스 채널 생성 - 기본 설정

JMS 엔진 설정의 서비스 채널 페이지에서 새 서비스 채널 생성 링크를 클릭하여 이 페이지로 이동할 수 있다.

우선 새로 추가할 서비스 채널의 이름을 Name 필드에 입력한다. 이 이름은 JMS 커넥션 팩토리를 설정할 때 사용된다.

다음으로 서비스를 제공할 주소를 지정해야 하는데, jms-tcp-address와 server-url이라는 두 가지 Address Type 중 어느 것을 선택하느냐에 따라 입력 방식이 다르다.

jms-tcp-address 방식

Server Address 필드에 IP 주소를, Port 필드에 TCP 포트 번호를 입력한다. 포트 번호가 0이거나 지정되지 않으면 JEUS 엔진 컨테이너의 Base Port를 통해 서비스 된다.

server-url 방식

Server Url 필드에 서비스를 제공할 URL을 입력한다. TCP 프로토콜을 사용하는 경우 tcp://HOST:PORT의 형식으로 입력한다. 또는 JEUS 엔진 컨테이너의 Base Port를 통해 서비스 하기 위해 oneport라고 설정한다.

참고

oneport는 Base Port를 통해 JNDI, JMS 등 여러 가지 서비스를 제공할 수 있게 해주는 JEUS의 포트 통합 기능이다.

추가로 다음 고급 선택사항을 설정할 수 있다.

Client Limit

이 서비스 채널로 접속할 수 있는 최대 클라이언트 수를 설정한다. 기본값으로 1000이 지정되어 있다.

Backlog Size

서비스 채널의 backlog 크기를 지정한다. 기본값으로 100이 지정되어 있다.

Blocking Socket

체크하면 서비스 채널이 Blocking 소켓을 사용하여 클라이언트와 통신하게 된다. 기본 설정으로 Blocking 소켓을 사용하지만 다수의 클라이언트가 접속했을 때 서버에 걸리는 부하를 줄이기 위해서는 Non-Blocking 소켓을 사용하는 것을 권장한다.

Read Buffer Size, Write Buffer Size

새로 추가할 서비스 채널이 사용할 Read/Write 버퍼의 크기를 바이트 단위로 설정한다. 기본값으로 65536(64KB)이 지정되어 있다.

Connection Timeout

지정한 시간 동안 아무런 요청이 없는 클라이언트와의 연결을 종료하도록 설정한다. 기본값으로 120분이 지정되어 있다.

Client Keepalive Timeout

클라이언트와의 연결이 비정상적으로 종료됐을 때 클라이언트가 커넥션 Failover를 시도하여 다시 연결될 경우를 위해 재접속을 기다리는 시간을 설정한다. 이 기간 동안 클라이언트가 생성한 서버 측 리소스는 해제되지 않는다. 기본값으로 30초가 지정되어 있다.

Check Security

이 필드에 체크하고 서비스 채널을 생성하면 다음과 같은 보안 도메인의 정책 설정이 적용된다.

  • jeus.jms.client.connectionFactory Permission을 갖고 createConnection Action이 허용되지 않은 사용자는 커넥션을 생성할 수 없다.

  • jeus.jms.destination.creation Permission을 갖고 createDestination Action이 허용되지 않은 사용자는 동적으로 데스티네이션을 생성할 수 없다.

Backup Service URL

Failover 설정시에 Backup으로 동작할 서버의 Service Channel의 URL을 설정한다. 자신이 Backup서버일 경우에는 Active의 Service Channel의 URL을 설정한다.

기본 설정을 마친 뒤 다음 버튼을 누르면 SSL 설정 페이지로 이동한다.

새 서비스 채널 생성 - SSL 설정

SSL 기능을 사용하려면 우선 Use SSL 필드를 체크한다. 그러면 SSL에 관한 다음과 같은 추가적인 설정을 할 수 있다.

Protocol

SSL에 사용할 통신 프로토콜을 설정한다. 기본값으로 TLS가 지정되어 있다.

Key Store Type

SSL 키 스토어의 종류를 설정한다. 기본값으로 JKS가 지정되어 있다.

Key Store File

키 스토어 파일의 절대 경로를 입력한다.

Key Store Pass

키 스토어 파일에 접근하기 위한 비밀번호를 입력한다.

Key Managerment Algorithm

키의 관리에 사용하는 알고리즘을 설정한다. 기본값으로 SunX509가 지정되어 있다.

Client Auth 필드를 체크하면 SSL 클라이언트의 인증에 필요한 다음과 같은 추가 설정을 할 수 있다.

Trust Store Type

SSL 클라이언트를 인증할 trust 스토어의 종류를 설정한다. 기본값으로 JKS가 지정되어 있다.

Trust Store File

trust 스토어 파일의 절대 경로를 입력한다.

Trust Store Pass

trust 스토어 파일에 접근하기 위한 비밀번호를 입력한다.

Trust Management Algorithm

trust의 관리에 사용하는 알고리즘을 설정한다. 기본값으로 SunX509가 지정되어 있다.

이전 버튼을 누르면 기본 설정 페이지로 이동한다.

모든 설정을 마친 뒤 생성 버튼을 누르면 JMS 엔진에 새 서비스 채널이 추가되면서 서비스 채널 페이지로 이동한다.

서비스 채널 - 설정

JMS 엔진 설정의 서비스 채널 페이지에 표시된 서비스 채널 목록에서 특정 서비스 채널의 이름을 클릭하면 이 페이지로 이동하여 설정 정보를 확인 및 수정할 수 있다.

여기에서는 서비스 채널의 이름(Name 필드)과 서비스 주소 유형(Address Type 필드)을 제외하고 서비스 채널을 생성할 때 입력했던 것과 같은 항목들을 설정할 수 있다. 각 항목에 대한 설명은 새 서비스 채널 생성 - 기본 설정 과 새 서비스 채널 생성 - SSL 설정 페이지의 도움말에 나와 있다.
