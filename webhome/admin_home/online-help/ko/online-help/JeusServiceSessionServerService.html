설정

기본 설정

중앙 세션 서버의 기본 설정을 정의한다. 중앙 세션 서버를 사용하기 위한 필수 항목(Required Field)은 없다.

Type

Primary가 디폴트이며 만약 Backup을 선택 한다면 다른 중앙 세션 서버 설정에 반드시 Primary가 존재 하여야 한다. 세션 클러스터링 된 환경에서는 Primary, Backup은 최대 1개씩만 존재 가능하다. 서블릿 엔진에서의 세션 컨테이너는 먼저 Primary 세션 서버와 연결되며 Primary 세션 서버가 down되었을 경우 Backup 세션 서버로 연결을 시도할 것이다.

Resolution

세션 서버 내에서, 세션 메모리에 있는 세션들에 대해 Passivation을 수행하거나 세션이 더 이상 유효하지 않을 때 세션들을 메모리에서 삭제하기 위해서는 자체 타이머가 필요하다. 이 Resolution값이 자체 타이머의 체크 주기에 해당하며 아래의 Passivation 타임아웃 또는 Removal 타임아웃을 이 값의 주기 동안 반복적으로 체크한다.

Connect Timeout

크게 두가지의 의미를 내포한다. 첫째는 중앙 세션 서버간의 연결을 맺을 때의 타임아웃, 둘째는 중앙 세션 서버의 Type이 Primary일 경우, 중앙 세션 서버와 서블릿 엔진의 세션 컨테이너간의 연결을 맺을 때의 타임아웃이다.

Read Timeout

크게 두가지의 의미를 내포한다. 첫째는 중앙 세션 서버간의 통신시 응답 메세지의 타임아웃, 둘째는 중앙 세션 서버의 Type이 Primary일 경우, 중앙 세션 서버와 서블릿 엔진의 세션 컨테이너간의 응답 메세지의 타임아웃이다.

Passivation To

중앙 세션 서버를 사용하면 세션이 모두 세션 서버 메모리에 집중되므로 JEUS 매니저가 과도한 메모리를 사용할 수 있다. 따라서 이 Passivation 타임아웃을 이용하여 특정 주기 동안 사용되지 않는 세션에 대해 파일 스토리지로 세션을 옮기는 작업을 수행하여야 한다.

Removal To

Removal 타임아웃 설정에 의해 파일 스토리지의 세션들을 제거할 수 있으며, 특히 Passivation 타임아웃을 사용하지 않을 경우, 중앙 세션 서버 입장에서는 세션들의 LifeCycle을 자체적으로 결정할 수 없으므로 이 Removal 타임아웃에 의해 메모리 세션들을 강제 제거 한다.

Check To

지정된 Check 타임아웃 주기마다 업데이트 대상의 세션들을 다른 중앙 세션 서버로 백업한다. 이 주기 값이 짧을 수록 현 중앙 세션 서버와 다른 중앙 세션 서버와의 세션 동기화가 잘 이루어지지만 업데이트 대상의 세션들이 자주 발생할 경우 네트워크에 부하가 걸릴 수 있다.

Backup Trigger

다른 중앙 세션 서버로 업데이트 대상의 세션들을 백업할 때 과도한 네트워크의 부하를 줄이기 위해 특정 갯수의 세션이 모일 때까지 버퍼링 하여 한번에 전송하도록 한다. 이때 최대 몇 개의 세션들을 버퍼링 할지의 갯수를 설정한다.

Check Level

업데이트 대상의 세션들을 정의하기 위한 레벨을 선택한다. 보통 set으로도 충분 하나, 항상 모든 세션을 다른 중앙 세션 서버와 실시간으로 동기화시키고 싶을 경우 all을 사용하도록 한다.

Recovery Mode

세션 서버가 다운 되었다가 다시 부트될 때, 다른 세션 서버로부터의 복구 모드를 선택한다. 보통 active모드만으로도 정상 동작에 충분하다.

쓰레드 풀

중앙 세션 서버는 여러 서블릿의 세션 컨테이너들과 동시 접속을 맺어 많은 세션을 저장, 관리한다. 이때 많은 세션들이 동시에 중앙으로 집중될 수 있기 때문에 특별히 쓰레드들을 풀링하여 관리한다. 동시 Connection의 상황, 세션 데이타의 전송 양, 크기에 따라 요청 처리 쓰레드 풀값을 조절하도록 한다.

Min

최소의 쓰레드 풀링 갯수를 설정한다.

Max

최대의 쓰레드 풀링 갯수를 설정한다.

Period

풀링 쓰레드가 정리되는 주기를 설정한다.

파일 스토리지

중앙 세션 서버는 여러 서블릿의 세션 컨테이너들과의 동시 접속을 맺어 많은 세션들을 메모리에 저장, 관리한다. 이때 많은 세션들이 중앙에 집중될 경우 JEUS 매니저에 과도한 메모리 부하가 걸릴 수 있다. 이를 방지하기 위해 Passivation 정책을 통해 메모리 세션들을 비활성화 상태로 바꾸어 파일 스토리지에 저장한다. 파일 스토리지를 사용할 경우 세부 항목들을 설정하기 위한 값들이다.

Min Hole

설정된 값 이상의 파일 IO가 일어날 경우 파일 스토리지를 재 정리하여 파일 스토리지의 크기를 최적화 시킨다.

Packing Rate

먼저 Packing Rate을 계산한다. Packing Rate이란 (파일 IO횟수 / 현 세션의 수)를 의미하며 소수점을 가진 값이 된다. 설정된 값의 비율이 이 설정한 Packing Rate 이상이 되었을 경우 파일 스토리지를 재 정리하여 파일 스토리지의 크기를 최적화 시킨다.

File Db Path

파일 스토리지의 절대 경로를 지정한다. 이 설정은 파일 스토리지를 사용하기 위한 디렉토리를 의미한다.

File Db Name

파일 스토리지의 파일 이름을 지정한다. 위의 File Db Path에 설정한 디렉토리 밑에 이 값의 이름으로 파일 스토리지가 생성된다.

통계

JEUS 매니저에 존재하는 중앙 세션 서버에 대한 통계치이다. 중앙 세션 서버가 현 JEUS 매니저에 존재하여야만 통계 자료가 나타난다.

메모리(활성) 세션 갯수

세션 서버의 현재 활성화된 세션의 갯수를 의미한다. 즉, JEUS 매니저 로컬 메모리에 저장되어 있는 세션의 갯수이다.

파일 스토리지(비활성) 세션 갯수

세션 서버의 현재 비활성화된 세션의 갯수를 의미한다. 즉, 세션 서버의 파일 스토리지에 저장되어 있는 세션의 갯수이다.

현재의 세션 메모리 크기

세션 서버의 현재 메모리에 저장되어 있는 활성화된 세션의 총 크기를 대략 적으로 나타낸다. 단위는 byte이다.

최고 상태의 세션 메모리 크기

세션 저장 메모리가 최대치였을 때의 값을 의미한다. 단위는 byte이다.

1분간의 세션 메모리 변화량

세션 저장 메모리의 1분간의 변화량을 의미한다. 메모리가 줄었을 경우 음수로, 늘었을 경우 양수로 나타나며, 변화가 없을 시 0이다. 단위는 byte이다.

5분간의 세션 메모리 변화량

세션 저장 메모리의 5분간의 변화량을 의미한다. 메모리가 줄었을 경우 음수로, 늘었을 경우 양수로 나타나며, 변화가 없을 시 0이다. 단위는 byte이다.

15분간의 세션 메모리 변화량

세션 저장 메모리의 15분간의 변화량을 의미한다. 메모리가 줄었을 경우 음수로, 늘었을 경우 양수로 나타나며, 변화가 없을 시 0이다. 단위는 byte이다.
