서블릿 엔진

네비게이션 경로: Home > node-name > 엔진 컨테이너 > container-name > 엔진 > 서블릿엔진

개요

이 페이지에서는 서블릿엔진의 각종 정보를 설정할 수 있다. 그리고 컨텍스트 그룹 탭을 클릭하면 서블릿엔진에 설정되어 있는 컨텍스트 그룹의 목록을 표의 형태로 볼 수 있다. 컨텍스트 그룹에 원하는 명령(정지, 계속, 종료, 시작)을 내릴 수 있으며 새 컨텍스트 그룹을 생성, 배포할 수 있다. 또한 설정을 다른 서블릿 엔진으로 배포할 수 있다.

각각에 대해 아래 절에서 설명한다.

설정

서블릿엔진에서 필수적으로 설정해야 할 사항은 없다. 아래 도움말을 참고하여 필요한 항목만 설정하도록 한다.

1. 기본 설정

기본 설정에는 다음 네 개의 설정이 있다.

Redirect Stdout

Web Container에 대한 System의 표준 출력을 다음 경로 JEUS_HOME\logs\JeusSystem\[node-name]_[container-name]\stdout_[date].log 파일로 저장 가능하다.

Redirect Stderr

Web Container에 대한 System의 표준 에러 출력내용 다음 경로 JEUS_HOME\logs\JeusSystem\[node-name]_[container-name]\stderr_[date].log 파일로 저장 가능하다.

Shutdown Timeout

이 설정은 관리자가 "down" 명령을 내렸을 때 Web Container가 실제로 종료되기까지 기다리는 시간을 millisecond단위로 지정한다. 즉 Web Container가 worker thread들의 작업을 모두 마치기까지 기다리도록 관리자가 설정가능하다. "down" 명령을 수행했을 때 실행중인 worker thread가 없으면 Web Container는 설정을 무시하고 종료된다.

Property

서블릿엔진 전체에 적용되는 property 를 설정한다.

2. 에러 로그

에러 로그를 설정하는 곳으로 다음과 같은 항목이 있다.

Name

로그의 이름을 설정한다.

Level

logging의 level을 설정한다. 각 level의 의미는 J2SE의 logging API의 Level class를 참고하기 바란다.

Use Parent Handlers

parent logger의 handler들을 이 logger에서도 사용할지를 결정한다.

Filter Class

logger에 지정할 filer class의 fully qualified class name을 설정한다.

3. 액세스 로그

액세스 로그를 설정하는 곳으로 다음과 같은 항목이 있다.

Enable

엑세스 로깅 기능의 사용 여부를 설정한다.

Enable Analysis

엑세스 로깅 분석 여부를 설정한다.

Format

time-stamp를 위한 포맷을 설정한다. 이 설정은 엑세스 로그에서만 사용한다.

Exclude Ext

access-log를 남기지 않는 확장자를 설정한다.

Name

로그의 이름을 설정한다.

Use Parent Handlers

parent logger의 handler들을 이 logger에서도 사용할지를 결정한다.

Filter Class

logger에 지정할 filer class의 fully qualified class name을 설정한다.

4. 사용자 로그

로그의 이름을 설정한다.

Level

Use Parent Handlers

5. 모니터링

Web Container의 여러 모니터링 thread의 환경설정을 위한 페이지이다. 이 설정에 따라 모니터링 thread가 일정 간격으로 여러 pool이나 자원의 상태를 점검하게 된다. 아래와 같은 세 가지 설정이 있다.

Check Thread Pool

worker 쓰레드를 체크하는 시간 간격이다. 시간간격은 millisecond단위(가장 근접한 초의 반올림이 된다) 가능한 가장 짧은 주기는1000 millisecond, 즉 1초이다.

Check Class Reload

변경된 클래스를 리로딩 하기 위해 체크하는 시간 간격이다. 단 <check-on-demand> 가 설정되어 있으면 주기적으로 점검하는 대상에서 제외된다. 단위는 역시 millisecond 이며 -1로 설정하면 비활성화 된다.

Check Session

세션 타임아웃을 점검하기 위한 주기이다. millisecond 단위로 설정하며 -1 로 설정할 경우 비활성화 되어 HTTPSession.invalidate()가 발생하기 전까지는 모든 세션이 보존된다.

6. 세션

세션 서브탭에는 기본으로 다음과 같은 설정이 있다.

Distributable

세션 클러스터링의 참여 여부 결정한다. 세션 클러스터링의 종류에는 중앙식 세션 클러스터링과 분산식 세션 클러스터링이 있다. 세션 클러스터링의 종류는 JEUSMain.xml의 설정에 따라 결정된다. 세션 클러스터링에 참여하기 위해서는 JEUSMain.xml에 중앙식 또는 분산식의 세션매니저 설정이 있어야 하며, 노드 클러스터링을 전제로 한다. 이 값이 true라 하더라도 JEUSMain.xml에 세션 매니저 설정이 없다면 세션 클러스터링에 참여하지 않는다. 설정값은 Boolean값(true/false)이다. 디폴트 값은 false.

Shared

Context간에 세션의 공유 여부를 결정한다. 세션 공유는 한 Context에서 생성된 세션 객체를 다른 Context에서도 사용할 수 있는 기능을 말한다. 세션 공유를 활성화 시키고 세션 클러스터링에 참여하지 않았을 경우, 세션 객체의 공유 범위는 어플리케이션 단위이다. 즉, 같은 어플리케이션 안에서는 같은 세션 객체를 공유할 수 있다. 설정값은 Boolean값이다. 디폴트 값은 false이다.

Timeout

세션 객체가 가장 최근에 사용된 이후의 유효기간을 결정한다. 유효기간 이후의 세션은 제거된다. web.xml에 세션 타임아웃 값이 설정되었을 경우 이 설정값보다 우선 적용된다. 시간 단위는 분 단위이다. 디폴트 값은 30분이다. -1로 설정하면 이 세션의 유효기간은 무한대이다.

Reload Persistent

Context가 reload될 때 세션의 attribute객체들을 계속 유지할지 여부를 결정한다. true일 경우 reload와 관계없이 세션의 attribute들은 계속 유지되며, false일 경우 reload시 모든 attribute들이 제거된다. 설정값은 Boolean값이다. 디폴트 값으로 false이다.

Url Rewriting

Web Container는 클라이언트로부터 요청에 대한 세션을 유지하기 위해서 기본적으로 쿠키를 통해서 세션 ID를 주고 받는다. 하지만, 쿠키가 처음 만들어진 곳의 도메인 이름과 다른 도메인 이름을 가진 요청을 보낼 때 브라우저는 쿠키로 세션 ID를 전달하지 않는다. 그래서 다른 도메인 이름을 가진 일련의 요청에 같은 세션 ID를 사용해야 할 때 이 옵션을 사용한다. 이 옵션을 true로 설정하면 비록 웹 브라우저가 쿠키를 지원하더라도 쿠키로 세션 정보를 전달하는 대신 URL rewriting을 통해서 세션 ID를 전달하게 한다. 이 경우 비록 다른 도메인 이름을 통해 여러 호출이 사용되어도 세션 ID의 교환은 이루어질 것이다. 이것은 context에 의해서 반환되는 모든 URL에 유일한 JSESSIONID가 URL파라미터로 붙게 된다. 만일 false로 설정되면 Url rewriting은 사용되지 않을 것이고 디폴트 쿠키가 기준으로 사용된다. 설정값은 Boolean값이다. 디폴트 값은 false이다.

Jsessionid Name

세션 ID 이름으로 표준 이름인 “JSESSIONID” 대신에 다른 이름을 사용하고자 할 때 이 설정을 사용한다. 디폴트 값은 "JSESSIONID"이다.

Version

쿠키 ID 버전을 설정한다. 설정값은 0 혹은 1이다. 0이면 디폴트 NS 쿠키 유형, 1이면 RFC 스펙의 기본 쿠키 유형을 의미한다.

Domain

Session id 쿠키를 보내는 서버의 도메인 이름을 설정한다. cookie는 이 도메인 요청에 대해서만 되돌아 온다. 하나의 적합한 도메인 이름. 이는 “.”으로 시작되고 hostname을 지정해선 안 된다(RFC-2109 스펙에 의해).

Path

하나의 session id 쿠키가 보내질 도메인 내에 URL 경로를 설정한다. 쿠키는 도메인이 적합할 때까지 해당 URL에 요청과 더불어 보내진다(<domain>element를 참조). 예를 들어 만일 “/examples”란 경로가 설정되고, 도메인은 “.foo.com”으로 설정되고 클라이언트 요청들은 "www.foo.com/examples" 이면 클라이언트의 쿠키는 요청과 더불어 보내진다. 도메인 밑에 하나의 적합한 URL경로 /로 설정하면 도메인이 적합할 동안 요청경로에 상관없이 쿠키는 보내진다.

Max Age

Session id 쿠키의 "expires" 속성을 설정한다. 이 시간주기가 되면 쿠키는 클라이언트로부터 제거되고 더 이상 보내지 않는다. 시간 단위는 초 단위이다. 디폴트 값은 -1이다. -1로 설정하면 해당 값은 쿠키의 "expires"속성을 사용하지 않는 것을 나타내고 따라서 쿠키는 사용자의 세션이 끝남과 동시에 종료된다.

Secure

Session id 쿠키의 “secure”속성을 설정한다. 설정값은 Boolean값이다. 만일 “true”로 설정되면 session id 쿠키는 오직 secure HTTPS 커넥션으로 보내진다. false 로 설정하면 커넥션 유형에 상관없이 session id 쿠키를 보낸다. 디폴트값은 false이다.

7. 세션 서버

이 element는 JEUS 노드 클러스터링 환경이 아닐때에, 웹컨테이너에서 중앙 세션 서버를 사용하고자 할 때 설정한다.

Primary Server

중앙 세션 서버를 사용하고자 할때 주 세션 서버 역할을 하는 곳의 JEUS 노드 이름을 설정한다. 이 노드에 대한 물리적인 host 정보는 vhost.properties에 구성되어 있어야 한다.

Backup Server

중앙 세션 서버를 사용하고자 할때 백업 세션 서버 역할을 하는 곳의 JEUS 노드 이름을 설정한다. 이 노드에 대한 물리적인 host 정보는 vhost.properties에 구성되어 있어야 한다. 만약 주 세션 서버가 다운된 상태에서 웹 컨테이너는 이 백업 세션 서버로 takeover한다.

Connect Timeout

JEUS 매니저에 존재하는 중앙 세션 서버와 웹 컨테이너 서블릿 엔진 사이의 통신에 있어 연결을 맺는데 기다리는 최대 시간을 나타낸다.

Read Timeout

JEUS 매니저에 존재하는 중앙 세션 서버와 웹 컨테이너 서블릿 엔진 사이의 통신에 있어 응답을 기다리는 최대시간을 나타낸다.

Check Level

웹컨테이너의 세션객체의 수정이 어느 정도 발생하였을 때 중앙 세션 서버로 세션을 업데이트 할지를 결정한다. all: getSession후에 세션 객체의 수정과 관련없이 모든 세션을 세션 서버로 업데이트한다. modified: getSession 후에 세션 객체의 수정 사항을 검사해서 변경시에 업데이트한다. set: 해당 세션의 setAttribute/putValue/removaAttribute/removeValue/setMaxInactiveInterval을 호출한 경우에만 세션을 업데이트한다.

컨텍스트 그룹 관리

컨텍스트 그룹 서브탭을 클릭하면 컨텍스트 그룹의 목록을 볼 수 있다. 이름을 클릭하면 해당 컨텍스트 그룹 정보 페이지로 이동한다. 명령에서는 컨텍스트 그룹이 시작이 안되었을 경우는 재시작 만이 가능하고 시작된 이후에는 계속, 종료, 재시작 (정지 상태의 경우), 정지, 종료, 재시작(계속 상태의 경우) 명령을 내릴 수 있다.

배포

다른 서블릿 엔진에 WEBMain.xml에 현재 서블릿 엔진의 WEBMain.xml을 적용한다.
