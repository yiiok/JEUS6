개요

이 페이지에서는 영구 모듈, 즉 JEUSMain.xml에 <application> 태그로 등록된 어플리케이션(모듈), 설정을 확인하고, 그 설정을 변경하거나 삭제할 수 있다.

이 페이지는 크게 두 개의 탭으로 구성되어 있다.

  1. 개요 : 현재의 어플리케이션 설정 내용을 xml 형식의 텍스트로 보여준다.

  2. 설정 : 현재의 어플리케이션에 대한 설정을 변경 및 삭제할 수 있다.

영구 모듈 - 어플리케이션

JEUSMain.xml에서 현재의 어플리케이션에 대한 설정 내용만을 xml 형식의 텍스트로 보여준다.

현재의 어플리케이션이 등록된 <application> 태그의 내용을 모두 보여주는 것이므로, 하나의 <application> 태그에 여러 개의 어플리케이션이 같은 설정으로 등록되어 있다면 등록된 모든 어플리케이션을 같이 출력한다.

영구 모듈 - 설정

현재의 어플리케이션에 대한 설정을 변경할 수 있다.

JEUSMain.xml에 <application> 태그로 등록된 각각의 어플리케이션을 테이블로 보여준다. 하나의 <application> 태그에 등록된 어플리케이션에 대한 정보가 하나의 테이블로 표시된다.

  1. 첫 번째 열에서는 <application> 설정 정보와 이 어플리케이션에 등록된 모듈의 종류를 보여준다. 각 아이콘이 나타내는 것은 아래와 같다.

    • : <application> 태그 내용

    • : Java EE 어플리케이션 (.ear 파일)

    • : EJB 모듈 (.jar 파일)

    • : 웹 어플리케이션 모듈 (.war 파일)

    • : 어플리케이션 클라이언트 모듈 (.jar 파일)

    • : 리소스 어댑터 모듈 (.rar 파일)

  2. 두 번째 열에서는 어플리케이션의 이름(모듈 이름)을 보여준다.

  3. 세 번째 열에서는 어플리케이션의 경로를 보여준다.

  4. 네 번째 열의 아이콘을 클릭하면 설정을 변경할 수 있는 페이지가 나온다.

    • 첫 줄에 나온 아이콘 클릭하면 이 어플리케이션에 공통으로 적용되는 설정들을 변경할 수 있는 페이지로 이동한다.

    • 각 어플리케이션 및 모듈에 해당하는 줄에 나온 아이콘을 클릭하면 해당 어플리케이션/모듈의 설정들을 변경할 수 있는 페이지로 이동한다.

  5. 마지막 열의 아이콘을 클릭하면 어플리케이션 및 모듈을 <application> 태그에서 삭제할 수 있다.

영구 모듈 디플로이 - 대상

여기에서는 어플리케이션을 디플로이 할 대상을 선택한다. 대상의 기본적인 단위는 엔진 컨테이너이므로 엔진 컨테이너 단위로 선택할 수 있다.

디플로이의 대상에는 다음의 3가지가 있다.

  • 현재 노드 : 현재 노드의 모든 엔진 컨테이너가 디플로이 대상이 될 수 있다.

  • 클러스터 : 현재 노드와 클러스터 된 모든 노드의 엔진 컨테이너가 디플로이 대상이 될 수 있다.

  • 가상 호스트 : 가상 호스트 내에 모든 엔진 컨테이너가 디플로이 대상이 될 수 있다. 가상 호스트 설정에 대한 내용은 가상 호스트를 참고하기 바란다.

웹 어플리케이션 모듈인 경우에는 컨텍스트 그룹과 가상 호스트를 설정할 수 있다(2. 가상 호스트 페이지 참고). 각 엔진 컨테이너 별로 서블릿 엔진의 설정 파일(WebMain.xml)에 설정된 컨텍스트 그룹과 가상 호스트 목록이 나타난다. 가상 호스트의 경우에는 설정이 되어 있지 않은 경우 'Default Virtual Host'가 목록에 나타난다.

가능한 대상에는 사용자가 선택할 수 있는 디플로이 대상이 나타나므로, 선택할 대상에 대해 선택 상자를 클릭하면 그 대상이 선택된다.

  • 모두 : 클러스터, 가상 호스트를 포함한 모든 가능한 노드 내에 있는 엔진 컨테이너가 모두 선택된다.

  • 노드 : 클러스터, 가상 호스트를 포함한 모든 가능한 노드를 표시한다. 아이콘과 함께 노드 이름이 표시된다.

  • 엔진 컨테이너 : 클러스터, 가상 호스트를 포함한 각 노드에 설정된 엔진 컨테이너를 표시한다. 아이콘과 함께 엔진 컨테이너 이름이 표시된다. 웹 어플리케이션 모듈인 경우에는 컨테이너 하위에 컨텍스트 그룹과 가상 호스트 설정 목록이 추가로 나타난다.

영구 모듈 디플로이 - 설정

어플리케이션에 공통으로 적용될 설정들을 변경할 수 있다.

Absolute Path

하위 어플리케이션이 존재하는 절대 경로를 지정한다. 지정하지 않을 경우 각 컨테이너의 APP_HOME에서 찾는다.

Fast Deploy

해당 어플리케이션 내의 EJB module, 웹 어플리케이션의 웹 서비스 모듈에 대해 패스트 디플로이(fast deploy)를 적용하도록 설정한다. 즉, 이 어플리케이션이 디플로이 과정에서 생성되어야 하는 클래스를 미리 포함하고 있다고 가정한다. 디플로이를 처음 하거나 모듈의 클래스 파일들이 변경되었을 경우에는 이 옵션을 사용하지 않는다. 그러나 appcompiler 툴을 사용하여 클래스들을 생성한 모듈의 경우에는 이 옵션을 사용할 수 있다.

Deploy When Booting

JEUS가 부팅될 때 해당 어플리케이션을 디플로이 할 것인지에 대한 설정을 한다. 기본값은 true이다.

Classloading

이 application이 사용할 classloading 방식을 선택한다. 지정하지 않으면 jeus.classloading system property에 설정되어 있는 값을 사용한다. JEUS 6에서는 ISOLATED 방식을 사용할 것을 적극 권장한다.

Class Ftp Unit

어플리케이션에 포함된 EJB 모듈의 클래스를 class ftp를 통해서 클라이언트(client)에게 전달할 때의 단위로 CLASS와 JAR 두가지 단위가 있다. 기본값은 JAR이다.

Security Domain Name

이 application에게 적용할 security domain을 설정한다. 지정하지 않으면 DEFAULT_APPLICATION_DOMAIN을 사용하게 된다.

Keep Generated

이 applicaton 내의 module들에 keep-generated를 적용한다. 즉, 이 appliation이 deploy과정에서 generated되어야 하는 클래스를 미리 포함하고 있다고 가정한다. 만약 설정이 되어 있지 않으면 jeus.application.keepgenerated system property에 지정된 값이 사용된다.

Auto Deploy

모듈 및 디렉토리에 대한 auto deploy 설정을 한다.

이 어플리케이션에 대해 archive 파일이나 Java EE 표준 DD가 변경되었을 경우 자동적으로 리디플로이(redeploy)를 실행한다.

Check Absolute Path

디렉토리에 대한 오토 디플로이 설정을 할 경우에만 사용된다. 어플리케이션에 대한 오토 디플로이 설정을 하는 경우에는 이 태그의 값을 false로 해야한다. 설정을 하지 않으면, 자동으로 false가 설정된다.

Auto Deploy Check Interval

JEUS 서버가 해당 경로에 있는 어플리케이션들의 변경 사항을 확인하기 주기를 설정한다. 기본값은 10초이며, 주기가 너무 짧을 경우 성능 저하가 있을 수 있다.

Java Security Permission

이 어플리케이션에 J2SE security permission을 설정할 수 있다.

Description

이 어플리케이션에 security permission 설정에 대한 설명을 적는다.

Security Permission Spec

이 어플리케이션에 적용할 security permission 설정을 기술한다. 형식은 java policy file과 동일하다.

영구 모듈 디플로이 - 컴포넌트별 선택 사항

어플리케이션에 포함된 각 컴포넌트에 대한 설정을 변경할 수 있다.

EJB 컴포넌트 선택 사항

이 어플리케이션 내 EJB 컴포넌트에 대한 특별한 설정을 하고 싶을때 사용한다.

Uri

어플리케이션 내에 있는 이 EJB 컴포넌트에 해당하는 uri를 설정한다. standalone 모듈인 경우에는 이 값을 설정할 필요가 없다.

WebAdmin에서는 이 값을 자동으로 설정해준다.

Client View Path

이 EJB 컴포넌트의 스텁 클래스들이 존재할 디렉토리를 설정한다.

Keep Generated

이 EJB 컴포넌트의 스텁 클래스들을 생성할때 자바 소스 파일을 남길지의 여부를 지정한다. 이 설정이 없으면 <application> 하위의 <keep-generated>설정을 사용한다.
