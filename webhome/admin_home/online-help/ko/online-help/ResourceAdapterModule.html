설정

이 페이지에서는 디플로이된 특정 리소스 어탭터 모듈의 설정을 확인 할 수 있다.

jeus-connector-dd.xml 서브탭과 ra.xml 서브탭에서는 디플로이에 적용된 세부적인 설정을 확인할 수 있다.

통계

이 페이지에서는 Inbound 리소스 어댑터의 Worker Pool 통계 정보를 보여준다.

평균 사용 시간

Inbound 메시지에 의해 Worker 쓰레드가 사용된 시간의 평균값.

평균 수락 시간

작업 요청이 이루어진 뒤 Worker Pool이 작업을 수락하기까지 걸린 시간의 평균값.

평균 거부 시간

작업 요청이 이루어진 뒤 Worker Pool이 작업을 거부하기까지 걸린 시간의 평균값.

평균 시작 시간

작업 요청이 이루어진 뒤 Worker 쓰레드가 할당되어 실행을 시작할 때까지 걸린 시간의 평균값(작업이 수락된 경우).

평균 완료 시간

작업 요청부터 완료까지 걸린 시간의 평균값(작업이 수락된 경우).

생성

Worker 쓰레드를 생성한 횟수.

풀링된 쓰레드

Worker Pool의 크기.

가용한 쓰레드

현재 Worker Pool 내에 사용 가능한 쓰레드의 개수.

대기중인 쓰레드

Worker Pool이 작업을 수락하기를 기다리고 있는 쓰레드의 개수.

재조정수

Worker Pool에 남아 있는 Worker 쓰레드 중 일정 시간 동안 사용되지 않아 소멸시킨 쓰레드의 개수.

쓰레드 획득 실패수

Worker Pool이 작업 요청을 거부한 횟수.

제어

시작

정지된 상태의 어플리케이션을 실행시킨다.

시작 명령은 모듈이 정지 상태에 있을 때 할 수 있는 명령으로 여러 컨테이너에 이 모듈이 디플로이 되어 있는 경우(클러스터링의 경우)에는 이 모듈이 정지 되어 있는 컨테이너 리스트만 보여진다. 이 컨테이너 리스트 중 이 모듈을 시작 시키고 싶은 컨테이너만을 선택한다. 가능한 대상을 엔진 컨테이너 단위로 선택할 수 있으며, 모두를 선택하면 모든 엔진 컨테이너가 선택되며, 노드를 선택하면 노드에 있는 모든 컨테이너가 선택된다. 시작 버튼을 누르면 실행된다.

현재 어플리케이션이 실행되고 있는 상태라면 "가능한 대상이 없습니다."라는 메세지가 출력된다.

정지

실행되고 있는 상태의 어플리케이션을 정지시킨다.

정지 명령은 모듈이 시작 상태에 있을 때 할 수 있는 명령으로 여러 컨테이너에 이 모듈이 디플로이 되어 있는 경우(클러스터링의 경우)에는 이 모듈이 시작 상태에 있는 컨테이너 리스트만 보여진다. 이 컨테이너 리스트 중 이 모듈을 정지 시키고 싶은 컨테이너만을 선택한다. 가능한 대상을 엔진 컨테이너 단위로 선택할 수 있으며, 모두를 선택하면 모든 엔진 컨테이너가 선택되며, 노드를 선택하면 노드에 있는 모든 컨테이너가 선택된다. 정지 버튼을 누르면 실행된다.

현재 어플리케이션이 정지상태라면 "가능한 대상이 없습니다."라는 메세지가 출력된다.

제거

해당 어플리케이션을 제거(undeploy)한다.

제거 명령은 모듈이 시작 상태와 정지상태에 있을 때 할 수 있는 명령으로 이 모듈이 디플로이 되어 있는 모든 컨테이너 리스트가 보여진다. 이 컨테이너 리스트 중 이 모듈을 제거 시키고 싶은 컨테이너만을 선택한다. 가능한 대상을 엔진 컨테이너 단위로 선택할 수 있으며, 모두를 선택하면 모든 엔진 컨테이너가 선택되며, 노드를 선택하면 노드에 있는 모든 컨테이너가 선택된다. 제거 버튼을 누르면 실행된다.

리디플로이

이 모듈을 제거하고 다시 디플로이하는 명령이다. 단, 리디플로이 대상은 JEUS 어플리케이션 홈 (C:\JEUS\webhome\app_home)에 있다고 가정한다.

리디플로이 명령은 제거 명령과 같이 모듈이 시작 상태와 정지상태에 있을 때 할 수 있는 명령으로 이 모듈이 디플로이 되어 있는 모든 컨테이너 리스트가 보여진다.이 컨테이너 리스트 중 이 모듈을 제거하고 다시 디플로이 시키고 싶은 컨테이너만을 선택한다. 가능한 대상을 엔진 컨테이너 단위로 선택할 수 있으며, 모두를 선택하면 모든 엔진 컨테이너가 선택되며, 노드를 선택하면 노드에 있는 모든 컨테이너가 선택된다. 리디플로이 버튼을 누르면 실행된다.
