로거

개요

JEUS 시스템에서의 로그를 기록하는 로거를 생성/삭제하고, 설정을 변경하는 작업을 하는 페이지이다.

로깅 - 설정

이미 설정된 로거에 대한 설정을 변경하는 페이지이다. 로거의 설정 변경 및 로거가 가지고 있는 핸들러의 설정을 변경, 추가, 삭제할 수 있다.

로거 설정

로거의 설정을 변경한다.

Name

현재 설정을 변경할 로거의 이름을 나타낸다. 설정을 변경할 로거를 선택하여 이 페이지에 들어온 것이므로, 이 로거의 이름은 변경할 수 없다.

Level

로거의 로깅 레벨을 설정한다. 로깅 레벨은 아래와 같이 12개가 있다.

레 벨설 명
FATALJEUS 4.x대의 logger의 level로 compatibility를 위해 지원한다.SEVERE에 해당하는 레벨
NOTICEWARNING에 해당하는 레벨
INFORMATIONINFO에 해당하는 레벨
DEBUGFINE에 해당하는 레벨
SEVERE치명적인 실패, 오류(serious failure)에 대한 로깅 레벨
WARNING잠재적인 문제(potential problem)에 대한 로깅 레벨
INFO안내 메세지(information message)에 대한 로깅 레벨
CONFIG설정 메세지(static configuration message)에 대한 로깅 레벨
FINE트레이스(trace) 정보에 대한 로깅 레벨
FINERFINE 레벨보다 좀더 세세한 트레이스(trace) 정보에 대한 로깅 레벨
FINEST가장 세세한 트레이스(trace) 정보에 대한 로깅 레벨
ALL모든 메세지에 대한 로깅 레벨

참고

각 레벨에 대한 자세한 설명은 J2SE Logging API의 Level 클래스나 Javadoc을 참고하기 바란다.

고급 선택사항
Use Parent Handlers

이 로거에서 상위 로거(parent logger)의 핸들러들을 사용할지를 설정한다. 기본 값은 true로 상위의 handler를 사용한다. 하지만 "jeus" 로거의 경우는 JEUS의 최상위 로거이기 때문에 시스템 내부적으로 무조건 false로 동작한다.

Filter Class

이 로거에서 사용할 필터 클래스의 이름(fully qualified class name)을 설정한다.

로그 핸들러

등록된 로그 핸들러의 설정을 변경/삭제할 수 있으며, 새로운 핸들러를 생성할 수 있다.

등록된 로그 핸들러의 정보는 표로 나타나며 이름, 종류, 수준(레벨), 인코딩, 필터 클래스의 정보를 표시한다. 이름을 클릭하면 로그 핸들러 설정을 변경할 수 있는 페이지(로그 핸들러 설정)로 이동하며, 마지막 컬럼에 있는 아이콘을 누르면 등록된 핸들러를 삭제할 것인지에 대한 확인 메세지가 출력된 후 '확인'을 누르면 삭제된다.

로그 핸들러 테이블
이름

이 로거에 등록된 로거 핸들러의 이름을 나타내며, 이 로거 핸들러의 링크를 클릭하면 해당 로거 핸들러의 설정을 변경할 수 있는 페이지(로그 핸들러 설정)로 이동한다.

종류

이 로거 핸들러의 종류를 나타낸다.

핸들러의 종류는 다음과 같다.

  • Console : 콘솔 핸들러로, 로거를 통해 로깅되는 로그 메세지를 콘솔 화면에 출력한다.

  • File : 파일 핸들러로, 로거를 통해 로깅되는 로그 메세지를 지정한 파일로 저장한다.

  • Smtp : SMTP 핸들러로, 로거를 통해 로깅되는 로그 메세지를 이메일(e-mail)로 전송한다.

  • Socket : 소켓 핸들러로, 로거를 통해 로깅되는 로그 메세지를 소켓(socket)으로 전송한다.

  • User : 사용자 핸들러로, 로거를 통해 로깅되는 로그 메세지를 사용자가 만든 핸들러 클래스를 통해 처리한다.

수준

이 로거 핸들러의 로깅 레벨을 나타낸다. 로깅 레벨은 로거 설정의 LEVEL 설명을 참고하기 바란다.

인코딩

이 로거 핸들러에 로깅할 때 로그의 인코딩 방식을 나타낸다.

필터 클래스

이 로거 핸들러에서 로깅할 때 사용하는 필터의 클래스 이름을 나타낸다.

삭제()

표의 맨 마지막 컬럼 나타난 을 누르면 해당 로거 핸들러를 삭제할 것인지 확인 메세지가 출력되고, '확인'을 누르면 삭제된다.

새 콘솔 핸들러 생성

이 로거에 콘솔 핸들러를 추가한다. 콘솔 핸들러는 로거를 통해 로깅되는 로그 메세지를 콘솔 화면에 출력한다.

이 링크를 클릭하면 새 콘솔 핸들러 생성 페이지(새 로그 핸들러 생성 - 콘솔 핸들러)로 이동한다.

새 파일 핸들러 생성

이 로거에 파일 핸들러를 추가한다. 파일 핸들러는 로거를 통해 로깅되는 로그 메세지를 지정한 파일로 저장한다.

이 링크를 클릭하면 새 파일 핸들러 생성 페이지(새 로그 핸들러 생성 - 파일 핸들러)로 이동한다.

새 SMTP 핸들러 생성

이 로거에 SMTP 핸들러를 추가한다. SMTP 핸들러는 로거를 통해 로깅되는 로그 메세지를 이메일(e-mail)로 전송한다.

이 링크를 클릭하면 새 SMTP 핸들러 생성 페이지(새 로그 핸들러 생성 - SMTP 핸들러)로 이동한다.

새 소켓 핸들러 생성

이 로거에 소켓 핸들러를 추가한다. 소켓 핸들러는 로거를 통해 로깅되는 로그 메세지를 소켓(socket)으로 전송한다.

이 링크를 클릭하면 새 소켓 핸들러 생성 페이지(새 로그 핸들러 생성 - 소켓 핸들러)로 이동한다.

새 사용자 핸들러 생성

이 로거에 사용자 핸들러를 추가한다. 사용자 핸들러는 로거를 통해 로깅되는 로그 메세지를 사용자가 만든 핸들러 클래스를 통해 처리한다.

이 링크를 클릭하면 새 사용자 핸들러 생성 페이지(새 로그 핸들러 생성 - 사용자 핸들러)로 이동한다.

Name

이 로거에서 사용할 이름을 설정한다. 생략할 경우에는 이 로거가 노드나 엔진 컨테이너에 설정되는 경우에는 jeus logger이다. 엔진 아래에 설정되는 경우와 디폴트 컨테이너(default container)를 사용하여 엔진 컨테이너 아래에 설정되는 경우는 이 이름을 무시한다.
