설정

이 페이지에서는 디플로이된 특정 Java EE 어플리케이션의 설정을 확인 할 수 있다.

jeus-web-dd.xml 서브탭과 web.xml 서브탭에서는 디플로이에 적용된 세부적인 설정을 확인할 수 있다.

통계

통계 서브탭은 두 가지로 나뉜다. 먼저 기본에서는 어플리케이션이 어떤 컨텍스트 그룹에 등록되어 있는지와 총 요청이 들어온 개수와 평균 요청 처리 시간, 세션에 대한 정보를 볼 수 있다.

서블릿에서는 각각의 서블릿 별로 요청 수, 서블릿이 로딩되었는지 상태, 서블릿 클래스의 이름, jsp 의 경우 jsp 파일 이름, 웹서비스 여부를 표 형태로 볼 수 있다. 명령 란에서는 정지, 계속, 종료, 재시작 명령을 내릴 수 있다.

제어

시작

정지된 상태의 어플리케이션을 실행시킨다.

시작 명령은 모듈이 정지 상태에 있을 때 할 수 있는 명령으로 여러 컨테이너에 이 모듈이 디플로이 되어 있는 경우(클러스터링의 경우)에는 이 모듈이 정지 되어 있는 컨테이너 리스트만 보여진다. 이 컨테이너 리스트 중 이 모듈을 시작 시키고 싶은 컨테이너만을 선택한다. 가능한 대상을 엔진 컨테이너 단위로 선택할 수 있으며, 모두를 선택하면 모든 엔진 컨테이너가 선택되며, 노드를 선택하면 노드에 있는 모든 컨테이너가 선택된다. 시작 버튼을 누르면 실행된다. 현재 어플리케이션이 실행되고 있는 상태라면 "가능한 대상이 없습니다."라는 메세지가 출력된다.

정지

실행되고 있는 상태의 어플리케이션을 정지시킨다.

정지 명령은 모듈이 시작 상태에 있을 때 할 수 있는 명령으로 여러 컨테이너에 이 모듈이 디플로이 되어 있는 경우(클러스터링의 경우)에는 이 모듈이 시작 상태에 있는 컨테이너 리스트만 보여진다. 이 컨테이너 리스트 중 이 모듈을 정지 시키고 싶은 컨테이너만을 선택한다. 가능한 대상을 엔진 컨테이너 단위로 선택할 수 있으며, 모두를 선택하면 모든 엔진 컨테이너가 선택되며, 노드를 선택하면 노드에 있는 모든 컨테이너가 선택된다. 시작 버튼을 누르면 실행된다. 현재 어플리케이션이 실행되고 있는 상태라면 "가능한 대상이 없습니다."라는 메세지가 출력된다.

제거

해당 어플리케이션을 제거(undeploy)한다.

제거 명령은 모듈이 시작 상태와 정지상태에 있을 때 할 수 있는 명령으로 이 모듈이 디플로이 되어 있는 모든 컨테이너 리스트가 보여진다. 이 컨테이너 리스트 중 이 모듈을 제거 시키고 싶은 컨테이너만을 선택한다. 가능한 대상을 엔진 컨테이너 단위로 선택할 수 있으며, 모두를 선택하면 모든 엔진 컨테이너가 선택되며, 노드를 선택하면 노드에 있는 모든 컨테이너가 선택된다. 제거 버튼을 누르면 실행된다.

리디플로이

해당 어플리케이션을 제거(undeploy)하고 다시 디플로이한다. 단, 리디플로이 대상은 JEUS 어플리케이션 홈 (C:\JEUS\webhome\app_home)에 있다고 가정한다.

리디플로이 명령은 제거 명령과 같이 모듈이 시작 상태와 정지상태에 있을 때 할 수 있는 명령으로 이 모듈이 디플로이 되어 있는 모든 컨테이너 리스트가 보여진다.이 컨테이너 리스트 중 이 모듈을 제거하고 다시 디플로이 시키고 싶은 컨테이너만을 선택한다. 가능한 대상을 엔진 컨테이너 단위로 선택할 수 있으며, 모두를 선택하면 모든 엔진 컨테이너가 선택되며, 노드를 선택하면 노드에 있는 모든 컨테이너가 선택된다. 리디플로이 버튼을 누르면 실행된다.

리디플로이 과정에서 WebtobListener를 중지(Suspend)하고 다시 시작(Resume)에 대한 설정인 고급 선택 사항이 존재하며, DeployTarget의 ContextGroup 설정은 무시한다.

WTBListener Block

리디플로이하기 전에 컨테이너 별로 WTBListener를 중지하고 다시 시작하는 기능을 적용한다.

WTBListener Block Type

WTBListener의 중지하고 다시 시작하는 방식을 결정한다.

each

각각의 컨테이너 별로 리스너 중지 - 리디플로이 - 리스너 재시작 한다.

half

절반의 컨테이너를 리스너 중지 - 리디플로이 - 리스너 재시작 한 후, 나머지 컨테이너를 리스너 중지 - 리디플로이 - 리스너 재시작 한다.

1:overflow

1개의 컨테이너를 리스너 중지 - 리디플로이 - 리스너 재시작 한 후, 나머지 컨테이너를 리스너 중지 - 리디플로이 - 리스너 재시작 한다

all

모든 컨테이너를 리스너 중지 - 리디플로이 - 리스너 재시작 한다.

WTBListener Block Wait

WTBListener를 중지 후 일정 대기 시간을 설정한다. (초 단위)
